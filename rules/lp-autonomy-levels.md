# LP Risk → Autonomy Level Map

> *Last-validated*: 2026-05-15
> *Review cadence*: quarterly — drop if 6 months without a triggering incident
> *Origin*: <meta-repo>; complements the more general
> `risk-tier-self-disclose.md` (low/med/high) — this file maps the
> LP-specific surfaces (hooks, `.org` sources, submodule pointers)
> onto the 4-level autonomy spectrum so AI agents know when to stop
> and ask.

The 4-level autonomy spectrum:

- **L1 — Inline accept.** Token-by-token autocomplete. N/A here.
- **L2 — Supervised, every step.** Agent proposes; human approves
  before any tool call executes.
- **L3 — Supervised + verification-first.** Agent acts, but every
  change passes through `make check-structure` / `make tangle` and
  human review before commit.
- **L4 — Fire-and-review.** Agent can land changes; human reviews
  the aggregated diff afterwards.

## LP surfaces (meta-repo)

| Path / change                                                  | Level | Rationale |
|----------------------------------------------------------------|-------|-----------|
| `lp/<sub>/<x>.org` — Python (or TS) src blocks                 | L3    | Tangle output reaches a *downstream submodule*; verification via `make tangle FILE=…` + `git -C repos/<sub> diff --stat` empty |
| `lp/<sub>/<x>.org` — prose only (Verified-by, comments, NL outline) | L4 | No code change |
| `lp/<sub>/<x>.org` — heading promotion / demotion / restructure | L3   | INDEX, anchor drift; depth lint may fire |
| `repos/<sub>/<...>/*.py`                                       | **blocked** | PreToolUse hook rejects (except `alembic/versions/*.py`) |
| `repos/<sub>/` submodule pointer (`.gitmodules`, `git submodule update`) | **L2** | A pointer change pollutes every product-repo CI that consumes the submodule |
| `lp/draft.org`, `lp/decisions-log.org`                          | L4    | Prose archive |
| `INDEX.org`                                                    | L4    | Generated by `make build-index`; never edit by hand |
| `.claude/rules/*.md`                                           | L3    | Rules constrain *future* agent behavior; review carefully |
| `.claude/hooks/*.sh`                                           | **L2** | Hook misbehavior breaks the entire dev loop |
| `.claude/settings.local.json`                                  | **L2** | Hook registration; one wrong matcher and tooling stops |
| `.dir-locals.el`                                               | L3    | Affects every Emacs buffer in the repo (poly-org, save hooks) |
| `Makefile` — tangle / check-structure / build-index targets    | L3    | Migration plumbing; an off-by-one writes to the wrong path |
| `pyproject.toml`, `uv.lock` — new dependency                   | **L2** | Supply-chain risk; explicit approval per global rule |
| `scripts/check_org_structure.py` / `build_index.py` / `build_tangle_allowlist.py` | L4 | Self-contained lint / generator |
| `CLAUDE.md`                                                    | L4    | Prose only |

## Forced level changes

The strictest level wins when a change crosses surfaces:

- A change that touches a hook script AND a .org file runs at **L2**
  (the hook is the bottleneck).
- Any change that bumps a `repos/<sub>` pointer runs at **L2** —
  this is the meta-repo's single most consequential output.
- A change that adds a dependency to `pyproject.toml` runs at **L2**.
- A change that introduces a new `lp/<sub>/` directory (i.e. starts
  the migration of a new submodule) runs at **L3** — the agent must
  verify `make tangle` produces the expected files in `repos/<sub>/`
  before committing.

## What "verification-first" means at L3

Before commit, the agent runs (and the human checks):

```bash
make check-structure                     # depth ≤ 5, no grab-bag, prose-before-src
make build-index                         # regenerate lp/INDEX.org
make tangle FILE=lp/<sub>/<x>.org        # produce .py inside repos/<sub>/
git -C repos/<sub> diff --stat           # MUST be empty for an unchanged module
```

A non-empty submodule diff after tangling an *unchanged-intent* `.org`
means either:

1. The .org src block diverged from the .py source-of-truth (fix the
   .org), OR
2. Tangle behaviour changed (investigate before continuing).

Never paper over a non-empty tangle diff by re-running ruff format
until it "settles" — that hides drift.

## What "supervised every step" means at L2

Agent proposes a tool call (Bash, Edit, Write); human approves
*before* it executes. Claude Code's permission system supports this
via per-tool gating; L2 paths must never be in the auto-approve
allowlist of `.claude/settings.local.json`.

## When to revisit

Calibrated for the start of the meta-repo (Phase 0 done, no
submodules onboarded yet). Revisit when:

- Each submodule's first migration completes — confirm its `.py` row
  is now blocked by the hook (it already is by default — the hook
  blocks all `.py`).
- A specific path causes a real incident — promote one level, write
  a `tasks/lessons.md` entry explaining why.
- The first TS submodule onboards — extend this table with `.ts`
  rows and per-submodule formatter rows.
