* Introduction
I have used an org file to write my document and code for a long time, in various languages.

It is very convenient to write code in an org file.
For example, instead of managing multiple files in nested directories,
I can write code in an org file with a clear structure in a more lightweight way.
Also, it can contain both documentation and code in one file for all my projects.

Now I want to use org file to write Python code,
and I want to import Python modules from an org file for my projects.

So comes this library to help me import Python modules from org files in Emacs.

(defun literate-org-format-code-block ()
  (interactive)
  (when (org-in-block-p '("src"))
    (let* ((language (getf (second (org-element-at-point)) :language))
           (header-args (cl-third (org-babel-get-src-block-info 'light)))
           (file-name (cdr (assoc :tangle header-args))))
      (cond ((find language '("python" "ipython") :test #'equalp)
             (python-black-org-mode-block t))
            ((find language '("tsx" "ts" "js" "vue" "vue-html" "css") :test #'equalp)
             (literate-org-format-ts-at-point literate-org-ts-formatter))
            ((null language) (error "Unsupported language %s" language))))))

(defun literate-org-tangle-current-file ()
  (interactive)
  (when-let* ((header-args (cl-third (org-babel-get-src-block-info 'light)))
              (file-name (cdr (assoc :tangle header-args))))
    (if (and file-name (not (equal file-name "no")))
        (progn
          (org-babel-tangle '(16) file-name (f-ext file-name))
          (run-hook-with-args 'literate-org-after-tangle-current-file file-name))
      ;; Try to find the parent code block usng the current noweb-ref and tangle it.
      (when-let* ((noweb-ref (cdr (assoc :noweb-ref header-args))))
        (let ((noweb-ref-body (format "<<%s>>" noweb-ref)))
          (save-excursion
            (save-restriction
              (goto-char (point-min))
              (while (search-forward noweb-ref-body nil t)
                (when-let* ((header-args (cl-third (org-babel-get-src-block-info 'light)))
                            (file-name (cdr (assoc :tangle header-args)))
                            (actual-file-p (and file-name (not (equal file-name "no")))))
                  (org-babel-tangle '(16) file-name (f-ext file-name))
                  (run-hook-with-args 'literate-org-after-tangle-current-file file-name))))))))))

** tangle whole buffer on save

`literate-org-tangle-current-file' is point-based: it tangles only the
code block under point. That is correct for an interactive
"reformat-and-write-this-block" command (=M-x literate-org-tangle-current-file=
on a specific block) but wrong for the after-save-hook case.

When an external editor — Claude Code's =Edit= tool, an =emacsclient -e=
call, or any agent that mutates content outside the block at point —
triggers a save, point is unreliable. A point-based tangle silently
misses every change that did not happen at point. The fix is to tangle
*every* code block in the buffer; ~org-babel-tangle~ with no arguments
does exactly that, and is idempotent, so the worst case of running it
when nothing changed is one extra disk write of identical bytes.

This is the function bound to ~after-save-hook~ in ~literate-org-mode~:

#+BEGIN_SRC elisp
(defun literate-org-tangle-buffer ()
  "Tangle every code block in the current buffer, then format the .py
output directories via `literate-org-format-py'.

Per-block formatting via `python-black-org-mode-block' (in
`before-save-hook') is necessary but not sufficient: a formatter's
inter-definition rules — most visibly, two blank lines between
top-level defs — require seeing the *concatenated* tangle output, not
individual blocks. Running the formatter on the output directories
closes that gap; the formatter itself is selected by
`literate-org-py-formatter' (default `black', also `ruff' or nil).

We gate on the buffer's file extension rather than on `major-mode',
because `literate-elisp-load' temporarily switches the buffer to
`emacs-lisp-mode' during loading and a `derived-mode-p' check would
spuriously skip tangle in that window."
  (when (and (buffer-file-name)
             (string-suffix-p ".org" (buffer-file-name)))
    ;; Re-apply file-local variables before reading them. Two callers
    ;; would otherwise see stale values: `literate-org-tangle-by-path'
    ;; reverts with PRESERVE-MODES=t (skips `hack-local-variables'
    ;; intentionally to avoid restarting org-mode on every external
    ;; edit), and `save-buffer' never calls it. Without this hack-call
    ;; an updated `literate-org-py-formatter:' on the first line would
    ;; not take effect until the next full file re-visit.
    (hack-local-variables :no-mode)
    (let* ((inhibit-message t)
           (tangled (org-babel-tangle))
           (py-dirs (delete-dups
                     (mapcar #'file-name-directory
                             (seq-filter (lambda (f) (string-suffix-p ".py" f))
                                         tangled)))))
      (dolist (d py-dirs)
        (literate-org-format-py literate-org-py-formatter d)))))
#+END_SRC

** tangle by path (for external automation)

The companion entry point that automation hooks call when they have a
file path but no live buffer:

- =find-file-noselect= returns the existing buffer if the file is
  already open, otherwise loads it from disk.
- If the buffer is clean but disk has been modified externally (Claude
  wrote via =Edit=, or anything else changed it), revert the buffer so
  the tangle reflects the on-disk truth.
- If the buffer is dirty AND disk also changed externally, that is a
  genuine dual-writer conflict — signal an error and bail. The agent
  surfaces the message; the user resolves manually.

#+BEGIN_SRC elisp
(defun literate-org-tangle-by-path (path)
  "Load PATH into a buffer if needed and tangle.
Used by =.claude/hooks/tangle-org-buffer.sh= for the disk-write case.
Errors out if the buffer is dirty AND disk has changed externally —
that conflict requires human resolution."
  (with-current-buffer (find-file-noselect path t)
    (cond
      ((and (buffer-modified-p)
            (not (verify-visited-file-modtime (current-buffer))))
       (error "Buffer for %s has unsaved changes and disk has changed externally; resolve manually" path))
      ((not (verify-visited-file-modtime (current-buffer)))
       (revert-buffer t t t)
       (literate-org-tangle-buffer))
      (t
       (literate-org-tangle-buffer)))))
#+END_SRC

** Per-buffer formatter choice

After tangling, the =.py= output needs a formatter pass that sees the
*concatenated* file — Black/Ruff inter-definition rules (e.g. two
blank lines between top-level defs) cannot run per-block. This section
owns that pass and the knob that picks which formatter runs.

Selection is one buffer-local symbol. Three built-ins ship: =black=
(default; current behaviour), =ruff= (=ruff format=, fast Rust
formatter), and =nil= (skip — useful for debugging tangle output
verbatim). Override per file with the first-line file-local variable
form, or globally via =M-x customize-variable=:

#+begin_example
,# -*- mode: org; literate-org-py-formatter: ruff -*-
#+end_example

The dispatch uses =cl-defgeneric= rather than =pcase= — same
multimethod pattern already in =literate-org-node-name= (per-language
tree-sitter dispatch) and =inspector= (Python registry). New
formatters register without touching this section:

#+begin_example
;; in init.el or a plug-in literate-org file
(cl-defmethod literate-org-format-py ((_ (eql autopep8)) dir)
  (literate-org--run-formatter "autopep8" "--in-place" "--recursive" dir))
#+end_example

The =:safe= predicate is intentionally permissive (=symbolp=) so a
user-registered method is selectable via the file-local var without
extending an explicit allowlist. An unknown FORMATTER signals
=cl-no-applicable-method= at tangle time — explicit, not silent.

The =--run-formatter= helper wraps =call-process= in =ignore-errors=
because a missing formatter binary must not block =after-save-hook= —
the save still completes, the =.py= just lands unformatted. This is
the documented silent-failure exception per
=.claude/rules/code-quality-over-feature.md=: blocking saves on a
formatter outage punishes the wrong workflow.

#+BEGIN_SRC elisp
(defcustom literate-org-py-formatter 'black
  "Formatter applied to tangled Python output directories.

Built-in: `black' (default), `ruff', or nil to skip. Extend by adding
methods to `literate-org-format-py'. File-local override:

    # -*- mode: org; literate-org-py-formatter: ruff -*-"
  :type '(choice (const :tag "Black"          black)
                 (const :tag "Ruff format"    ruff)
                 (const :tag "None (skip)"    nil)
                 (symbol :tag "Custom (user-defined method)"))
  :group 'literate-org
  :safe #'symbolp)

(defun literate-org--run-formatter (cmd &rest args)
  "Run \"poetry run CMD ARGS...\" silently. Errors are swallowed: a
missing formatter must not break `after-save-hook'."
  (ignore-errors
    (apply #'call-process "poetry" nil nil nil "run" cmd args)))

(cl-defgeneric literate-org-format-py (formatter dir)
  "Format Python files in DIR using FORMATTER (a symbol).
Dispatch via `(eql FORMATTER)' specializer; an unknown FORMATTER
signals `cl-no-applicable-method'.")

(cl-defmethod literate-org-format-py ((_formatter (eql nil)) _dir)
  "Explicit opt-out; do nothing." nil)

(cl-defmethod literate-org-format-py ((_formatter (eql black)) dir)
  (literate-org--run-formatter "black" "--quiet" dir))

(cl-defmethod literate-org-format-py ((_formatter (eql ruff)) dir)
  (literate-org--run-formatter "ruff" "format" "--quiet" dir))
#+END_SRC

** xref backend
*** visit a point in the tangled file of a code block
#+BEGIN_SRC elisp
(defun literate-org-visit-point-at-tangled-position ()
  (when-let* ((header-args (cl-third (org-babel-get-src-block-info 'light)))
              (file-name (cdr (assoc :tangle header-args))))
    (when (file-exists-p file-name)
      (let* ((column-index (- (point) (line-beginning-position)))
             (last-code-block-line-no (save-excursion
                                        (search-backward-regexp "^#\\+BEGIN_SRC" nil t)
                                        (line-number-at-pos)))
             (line-index (- (line-number-at-pos)
                            last-code-block-line-no
                            1))
             ;; for each code block before the current code block for the same tangled file, count lines of each code block
             (lines-of-before-code-blocks nil))
        (save-excursion
          (search-backward (format ":header-args: :tangle %s" file-name) nil)
          (search-forward "#+BEGIN_SRC" nil)
          (let ((begin-line-no (line-number-at-pos)))
            (while (< begin-line-no last-code-block-line-no)
              (setf begin-line-no (line-number-at-pos))
              (search-forward "#+END_SRC" nil)
              (push (- (line-number-at-pos) begin-line-no 1)
                    lines-of-before-code-blocks)
              (search-forward "#+BEGIN_SRC" nil)
              (setf begin-line-no (line-number-at-pos)))))
        (setf lines-of-before-code-blocks (reverse lines-of-before-code-blocks))
        (with-current-buffer (find-file-noselect file-name)
          (goto-char (point-min))
          (while lines-of-before-code-blocks
            (forward-line (pop lines-of-before-code-blocks))
            (while (and (not (eobp))
                        (save-excursion
                          (beginning-of-line)
                          (looking-at-p "^\\s-*$")))
              (forward-line 1)))
          (forward-line line-index)
          (goto-char (line-beginning-position))
          (forward-char column-index)
          (values file-name (point)))))))
#+END_SRC


*** visit a point in a code block in the tangled file
#+BEGIN_SRC elisp
(cl-defun literate-org-point-at-code-block-of-tangled-location (org-buffer file-name point)
  (when (file-exists-p file-name)
    (with-current-buffer (find-file-noselect file-name)
      (save-excursion
        (goto-char point)
        (let* ((column-index (- (point) (line-beginning-position)))
               (line-index (line-number-at-pos))
               (org-point nil)
               (org-current-block-start-line nil)
               (org-current-block-lines 0))
          (goto-char (point-min))
          (with-current-buffer org-buffer
            (save-excursion
              (goto-char (point-min))
              (when (search-forward (format ":header-args: :tangle %s" file-name) nil t)
                (setf org-point (point)))))
          (when (null org-point)
            (cl-return-from literate-org-point-at-code-block-of-tangled-location nil))
          ;; while current block is not for current tangled line
          (while (<= (+ (line-number-at-pos) org-current-block-lines) line-index)
            ;; forward current block in tangled file.
            (forward-line org-current-block-lines)
            (while (and (not (eobp))
                        (save-excursion
                          (beginning-of-line)
                          (looking-at-p "^\\s-*$")))
              (forward-line 1))
            ;; search next code block in org buffer
            (with-current-buffer org-buffer
              (save-excursion
                (goto-char org-point)
                (search-forward "#+BEGIN_SRC" nil)
                (forward-line)
                (setf org-point (point))
                (setf org-current-block-start-line (line-number-at-pos))
                (search-forward "#+END_SRC" nil)
                (setf org-current-block-lines (- (line-number-at-pos) org-current-block-start-line)))))
          (let ((lines (- line-index (line-number-at-pos))))
            (with-current-buffer org-buffer
              (save-excursion
                (goto-char org-point)
                (forward-line lines)
                (goto-char (line-beginning-position))
                (forward-char column-index)
                (point-marker)))))))))
#+END_SRC

(defun literate-org-xref-show-definitions-buffer (fetcher alist)
  (let* ((xrefs (funcall fetcher))
         (current (current-buffer))
         (current-dir (when (buffer-file-name current)
                        (file-name-directory (buffer-file-name current))))
         ;; Search for org buffers in the same directory
         (org-bufs (cl-remove-if-not
                    (lambda (buf)
                      (with-current-buffer buf
                        (let ((buf-file (buffer-file-name)))
                          (and buf-file current-dir
                               (string= (file-name-directory buf-file)
                                        current-dir)))))
                    (org-buffer-list)))
         ;; put current org buffer in the first place
         (org-bufs (cons current (remove current org-bufs))))
    (loop for xref in xrefs
          for location = (xref-match-item-location xref)
          for group = (xref-location-group location)
          for (org-buffer . org-point-marker)
            = (loop for buf in org-bufs
                    for marker = (literate-org-point-at-code-block-of-tangled-location
                                          buf group (marker-position (xref-location-marker location)))
                    if marker
                      return (cons buf marker))
          if org-point-marker
            do (setf (xref-match-item-location xref)
                       (xref-make-buffer-location org-buffer (marker-position org-point-marker))))
    (let (buf)
      (cond
        ((not (cdr xrefs))
         (xref-pop-to-location (car xrefs)
                               (assoc-default 'display-action alist)))
        (t
         (setq buf
                 (xref--show-xref-buffer fetcher
                                         (cons (cons 'fetched-xrefs xrefs)
                                               alist)))
         (xref--auto-jump-first buf (assoc-default 'auto-jump alist))
         buf)))))
#+END_SRC

*** find definitions
We try to open the tangled file, and move cursor to the same point as it is in the code block,
then use lsp mode to find the definition, if the target position can be found in the current org file, we'll
move point to the target position.
#+BEGIN_SRC elisp
(defun literate-org-find-dwim ()
  (interactive)
  (unless (string= "org" (f-ext (buffer-file-name (current-buffer))))
    (lsp-find-definition)
    (return))

  (multiple-value-bind (tangled-file-name tangled-point)
      (literate-org-visit-point-at-tangled-position)
    (unless tangled-file-name
      (user-error "literate-org-find-dwim: no tangled file resolves at point"))
    (let ((loc
           (with-current-buffer (find-file-noselect tangled-file-name)
             (lsp 1)
             (goto-char tangled-point)
             (lsp-request "textDocument/definition" (lsp--text-document-position-params)))))
      (if (seq-empty-p loc)
        (lsp--error "Not found for: %s" (or (thing-at-point 'symbol t) ""))
        (let ((xref-show-definitions-function 'literate-org-xref-show-definitions-buffer))
          (lsp-show-xrefs (lsp--locations-to-xref-items loc) nil nil))))))
#+END_SRC

* Python library for a literate server
:PROPERTIES:
:header-args: :results silent :session :tangle no
:END:
** Idea

Python's import system is customisable via =sys.meta_path= — a
=MetaPathFinder= can satisfy =import foo= from any source the finder
chooses. We hand it org-section content so a single =literate-org.org=
serves as the package, no on-disk =.py= required at import time.
Per-block execution in the *target* module's namespace (instead of a
scratch REPL) keeps the org buffer and the live Python world in sync.

** Prototype
:PROPERTIES:
:header-args: :results silent :session :tangle no
:END:

End-to-end sanity check that =sys.modules= injection plus =exec= into
=module.__dict__= gives a real, importable module.

1. import the necessary modules
#+BEGIN_SRC python
import sys
import importlib
#+END_SRC

2. create a new module dynamically
#+BEGIN_SRC python
spec_module_a = importlib.util.spec_from_loader("module_a", loader=None)
module_a = importlib.util.module_from_spec(spec_module_a)
sys.modules["module_a"] = module_a
#+END_SRC

3. add a function to the module
#+BEGIN_SRC python
exec('def say_hello(): print("Hello from module_a.")', module_a.__dict__)
exec('def a(): return 3', module_a.__dict__)
exec('def b(): return 4 + a()', module_a.__dict__)
#+END_SRC
4. run this function
#+BEGIN_SRC python
from module_a import say_hello
from module_a import b
say_hello()
b()
#+END_SRC

** Module Layout in an Org file

Mapping from org structure to Python packages:

- Each org section with a =:LITERATE_ORG_MODULE:= property *is* a
  Python module; nesting depth mirrors dotted-module depth.
- Python =#+begin_src= blocks inside the section contribute its
  top-level defs, classes, and assignments.
- A block is loaded only when =LITERATE_ORG_LOAD= is unset, set to
  =yes=, or matched by the environment variable of the same name —
  the third case is how test-only blocks stay quiet during normal
  loading.

#+BEGIN_EXAMPLE
 * Module A
   :PROPERTIES:
   :LITERATE_ORG_MODULE: module_a
   :END:
  ** Module A.a
   :PROPERTIES:
   :LITERATE_ORG_MODULE: module_a.a
   :END:
  *** a function for module A.a
   ,#+BEGIN_SRC python
   def say_hello():

class LiterateImporter(object):
    def find_module(self, fullname: str, _path=None):
        if _get_module_spec(fullname):
            logger.debug(f"Found literate module {fullname}")
            return self
        else:
            return None

    def load_module(self, fullname: str):
        """Create a new module object."""
        mod_spec = _get_module_spec(fullname)
        mod = types.ModuleType(fullname)
        mod.__loader__ = self
        mod.__file__ = mod_spec.get("filepath", "")
        # Set module path - get filepath and keep only the path until filename
        mod.__path__ = ["/".join(mod.__file__.split("/")[:-1]) + "/"]
        mod.__package__ = fullname
        sys.modules[fullname] = mod
        # Execute the module/package code into the Module object
        logger.debug(f"Load literate module {fullname}")
        exec(mod_spec["content"], mod.__dict__)
        return mod

def optimal_clusters(definitions, embeddings, min_k=2, max_k=10, threshold=0.05):
    """
    Computes the optimal clusters for the given definitions and embeddings.

    It evaluates KMeans clustering for k in range(min_k, max_k+1) by computing the
    silhouette score for each k. If a candidate with more clusters has a silhouette score
    only marginally lower than the best score (within the threshold), it favors the higher k.

    Args:
        definitions: List of text definitions (e.g. function, class, or constant definitions).
        embeddings: Numpy array or list of embeddings corresponding to each definition.
        min_k: Minimum number of clusters to try.
        max_k: Maximum number of clusters to try.
        threshold: If the difference between the best score and a candidate score is less than
                   this threshold, choose the candidate with more clusters.

    Returns:
        clusters: A dictionary mapping each cluster label to a list of definitions.
        optimal_k: The optimal number of clusters chosen.
        scores: A dictionary mapping k to its silhouette score.
    """
    scores = {}
    # # Cap k at sample count and short-circuit the trivial "too few samples" case.
    # Limit max_k to the number of samples
    max_k = min(max_k, len(embeddings))

    # If too few samples, return one cluster containing all definitions.
    if len(embeddings) < 2:
        return {0: definitions}, 1, scores

    best_k = None
    best_score = -1

    # # Sweep k = min_k..max_k by silhouette score; ties within `threshold` favour the larger k.
    # Evaluate silhouette scores for each candidate k.
    for k in range(min_k, max_k + 1):
        kmeans = KMeans(n_clusters=k, random_state=42)
        labels = kmeans.fit_predict(embeddings)
        score = silhouette_score(embeddings, labels)
        scores[k] = score
        print(f"k={k}, silhouette score={score:.4f}")

        # For the first candidate, simply assign best_k.
        if best_k is None:
            best_k = k
            best_score = score
        else:
            # If a candidate has a slightly lower score (within threshold) but with more clusters,
            # favor it for increased granularity.
            if score > best_score:
                best_k = k
                best_score = score
            elif (best_score - score) < threshold and k > best_k:
                best_k = k
                best_score = score

    optimal_k = best_k
    print(f"\nOptimal number of clusters chosen: {optimal_k}")

    # # Re-run KMeans once more with the chosen k to get the final label assignments.
    # Run final clustering with the optimal number of clusters.
    final_kmeans = KMeans(n_clusters=optimal_k, random_state=42)
    final_labels = final_kmeans.fit_predict(embeddings)

    # # Group input definitions by their final cluster label and return the mapping.
    # Group definitions by cluster.
    clusters = {}
    for label, definition in zip(final_labels, definitions):
        clusters.setdefault(label, []).append(definition)

    return clusters, optimal_k, scores

def extract_imports_from_code(code: str, module_name: str) -> List[ImportData]:
    """Extract import statements from Python code."""
    imports = []
    # # Walk the AST; bail to an empty list on parse failure.
    try:
        tree = ast.parse(code)
        for node in ast.walk(tree):
            # # Plain `import X` / `import X as Y` — one entry per name in the import list.
            if isinstance(node, ast.Import):
                # Handle: import module_a, module_b as b
                for alias in node.names:
                    imports.append(ImportData(
                        module=alias.name,
                        definition=alias.asname or alias.name,
                        imported_symbol=None,
                        is_alias=alias.asname is not None,
                        is_wildcard=False
                    ))
            # # `from X import ...` — split the wildcard case from the named-symbol case.
            elif isinstance(node, ast.ImportFrom):
                if node.module:  # from module import ...
                    for alias in node.names:
                        if alias.name == '*':
                            # from module import *
                            imports.append(ImportData(
                                module=node.module,
                                definition='*',
                                imported_symbol=None,
                                is_alias=False,
                                is_wildcard=True
                            ))
                        else:
                            # from module import name [as alias]
                            imports.append(ImportData(
                                module=node.module,
                                definition=alias.asname or alias.name,
                                imported_symbol=f"{node.module}.{alias.name}",
                                is_alias=alias.asname is not None,
                                is_wildcard=False
                            ))
    # # Syntax-broken code is logged and treated as "no imports".
    except SyntaxError:
        logger.warning(f"Failed to parse imports from code in module {module_name}")
    return imports

class ModuleReloader:
    """Manages module dependencies and hot reloading."""
    
    def __init__(self):
        # Track module information
        self.modules: Dict[str, ModuleInfo] = {}
        # Track stale modules that need reloading
        self.stale_modules: Set[str] = set()
        # Object references for patching
        self.old_objects: Dict[Tuple[str, str], List[weakref.ref]] = {}
        
    def register_module(self, module_name: str) -> ModuleInfo:
        """Register a module if not already registered."""
        if module_name not in self.modules:
            self.modules[module_name] = ModuleInfo(name=module_name)
        return self.modules[module_name]
    
    def track_module_execution(self, module_name: str, code: str, 
                              top_level_names: Set[str]) -> Set[str]:
        """Track when a module is executed and return affected modules."""
        # Register this module
        module_info = self.register_module(module_name)
        
        # Extract imports from the code
        imports = extract_imports_from_code(code, module_name)
        module_info.imports = imports
        module_info.top_level_names = top_level_names
        module_info.last_modified = time.time()
        
        # Update imported_by relationships
        for import_data in imports:
            if import_data.module in self.modules:
                self.modules[import_data.module].imported_by.add(module_name)
        
        # Find modules that import from this module
        affected_modules = self.find_dependent_modules(module_name)
        
        # Mark them as stale
        self.stale_modules.update(affected_modules)
        
        return affected_modules
    
    def find_dependent_modules(self, module_name: str) -> Set[str]:
        """Find all modules that depend on the given module."""
        if module_name not in self.modules:
            return set()
        
        dependent = set()
        module_info = self.modules[module_name]
        
        # Direct dependencies
        dependent.update(module_info.imported_by)
        
        # Transitive dependencies (modules that import the dependent modules)
        to_check = list(dependent)
        checked = {module_name}
        
        while to_check:
            checking = to_check.pop()
            if checking in checked:
                continue
            checked.add(checking)
            
            if checking in self.modules:
                new_deps = self.modules[checking].imported_by
                for dep in new_deps:
                    if dep not in checked:
                        dependent.add(dep)
                        to_check.append(dep)
        
        return dependent
    
    def update_dependent_modules(self, source_module: str) -> Dict[str, List[str]]:
        """Update symbols in modules that import from source_module."""
        updates = {}

        # # Bail unless the source module is loaded *and* tracked by the reloader.
        if source_module not in sys.modules:
            return updates

        source_mod = sys.modules[source_module]
        module_info = self.modules.get(source_module)

        if not module_info:
            return updates

        # # For every module that imports from `source_module`, find its tracked metadata or skip.
        for dependent_module_name in module_info.imported_by:
            if dependent_module_name not in sys.modules:
                continue

            dependent_mod = sys.modules[dependent_module_name]
            dependent_info = self.modules.get(dependent_module_name)

            if not dependent_info:
                continue

            updated_names = []

            # # Walk that module's imports and dispatch on the import shape: wildcard / named / module-only.
            for import_data in dependent_info.imports:
                if import_data.module != source_module:
                    continue

                if import_data.is_wildcard:
                    # Handle from module import *
                    for name in module_info.top_level_names:
                        if hasattr(source_mod, name):
                            setattr(dependent_mod, name, getattr(source_mod, name))
                            updated_names.append(name)
                elif import_data.imported_symbol:
                    # Handle from module import name [as alias]
                    # Extract the actual name from imported_symbol
                    parts = import_data.imported_symbol.split('.')
                    if len(parts) > 1:
                        actual_name = parts[-1]
                        if hasattr(source_mod, actual_name):
                            # Update with the correct name (definition is the alias or original)
                            setattr(dependent_mod, import_data.definition,
                                   getattr(source_mod, actual_name))
                            updated_names.append(import_data.definition)
                else:
                    # Handle import module [as alias]
                    # Update the module reference itself
                    setattr(dependent_mod, import_data.definition, source_mod)
                    updated_names.append(import_data.definition)

            # # Record the rebound names per dependent module and return the full map.
            if updated_names:
                updates[dependent_module_name] = updated_names

        return updates
    
    def clear_stale_modules(self):
        """Clear the set of stale modules."""
        self.stale_modules.clear()
    
    def get_import_info(self, module_name: str) -> List[ImportData]:
        """Get import information for a module."""
        if module_name in self.modules:
            return self.modules[module_name].imports
        return []

def get_top_level_names(code):
    tree = ast.parse(code)
    variables = []
    functions = []
    classes = []

    for node in tree.body:
        if isinstance(node, ast.Assign):
            # Handle assignments like x = 1 or x, y = 2, 3.
            for target in node.targets:
                if isinstance(target, ast.Name):
                    variables.append(target.id)
                elif isinstance(target, ast.Tuple):
                    for elt in target.elts:
                        if isinstance(elt, ast.Name):
                            variables.append(elt.id)
        elif isinstance(node, ast.AnnAssign):
            # Handle annotated assignments like: x: int = 1.
            if isinstance(node.target, ast.Name):
                variables.append(node.target.id)
        elif isinstance(node, ast.FunctionDef):
            functions.append(node.name)
        elif isinstance(node, ast.ClassDef):
            classes.append(node.name)

    return variables, functions, classes

def ensure_module(module_name, module_create_method):
    """Ensure a module is loaded and return it."""
    if module_name in sys.modules:
        return sys.modules[module_name]

    match module_create_method:
        case "create":
            spec_module = importlib.util.spec_from_loader(module_name, loader=None)
            module = importlib.util.module_from_spec(spec_module)
            sys.modules[module_name] = module
            return module
        case "import":
            importlib.import_module(module_name)
            return sys.modules[module_name]
        case "import_or_create":
            if importlib.util.find_spec(module_name):
                importlib.import_module(module_name)
                return sys.modules[module_name]
            else:
                spec_module = importlib.util.spec_from_loader(module_name, loader=None)
                module = importlib.util.module_from_spec(spec_module)
                sys.modules[module_name] = module
                return module
        case _:
            msg = f"Module {module_create_method} doesn't exist"
            raise ValueError(msg)

def process_a_message(message):
    # # Capture buffers and the slots that will hold the response.
    stdout_stream = StringIO()
    stderr_stream = StringIO()
    error = None
    result = None
    locals = []
    updated_modules = {}  # Track modules updated by hot reload

    # # Inside stdout/stderr capture, parse the message and resolve the target module's namespace dict.
    with redirect_stdout(stdout_stream):
        with redirect_stderr(stderr_stream):
            try:
                type = message["type"]
                code = message["code"]
                dict = globals()
                module_name = message["module"] if "module" in message else None
                if module_name:
                    module_create_method = message.get("module-create-method", "import")
                    module = ensure_module(module_name, module_create_method)
                    dict = module.__dict__

                # # Dispatch on request type — `eval` and `exec` both run user code in the module namespace; only `exec` then resyncs server_locals to reflect any rebinding.
                if type == "eval":
                    exec(compile(code, module_name or "code", "exec"), dict)
                    message.get("result-name", "_")
                    result = dict.get("_", None)
                elif type == "exec":
                    result = exec(compile(code, module_name or "code", "exec"), dict)
                    vars_, funcs, classes = get_top_level_names(code)
                    all_top_level_names = set(vars_ + funcs + classes)

                    # Original server_locals tracking
                    for local in all_top_level_names:
                        if local in server_locals:
                            _local = server_locals[local]
                            if hasattr(_local, "__module__"):
                                _module_name = _local.__module__
                                if _module_name == module_name:
                                    # update the local with the new value
                                    server_locals[local] = getattr(module, local)
                                    locals.append(local)

                    # # Hot reload: track this module's execution and propagate new bindings to every module that imported it.
                    # Hot reload: Track module execution and update dependencies
                    if module_name:
                        # Track this module execution
                        module_reloader.track_module_execution(
                            module_name, code, all_top_level_names
                        )
                        
                        # Update dependent modules
                        updated_modules = module_reloader.update_dependent_modules(module_name)
                        
                        # Log what was updated
                        if updated_modules:
                            logger.debug(f"Hot reload updated modules: {updated_modules}")

                # # Other request shapes — `quit` clears the result; anything else is an error we re-raise.
                elif type == "quit":
                    result = None
                else:
                    error = "Unknown type: {}".format(type)
                    raise ValueError(error)
            except Exception as e:
                # printing stack trace
                traceback.print_exc()
                error = str(e)

    # # Build the response payload (success or error shape) and exit the process if `quit` was requested.
    if error is None:
        return_value = {
            "result": _inspect(result),
            "type": "result",
            "locals": locals,
            "stdout": stdout_stream.getvalue(),
            "stderr": stderr_stream.getvalue(),
        }
        # Add hot reload information if available
        if updated_modules:
            return_value["updated_modules"] = updated_modules
            return_value["stale_modules"] = list(module_reloader.stale_modules)
    else:
        return_value = {
            "error": error,
            "type": "error",
            "stdout": stdout_stream.getvalue(),
            "stderr": stderr_stream.getvalue(),
        }

    if type == "quit":
        sys.exit(0)
    else:
        return return_value

        code_a = """

    return 'a'

(defvar literate-org-last-response nil)

(cl-defun literate-org-request (path &key params data (type "GET"))
  (setf literate-org-last-response nil)
  (awhen (get-buffer "*literate-org-error*")
    ;; Close the buffer as it will be out of date.
    (kill-buffer it))
  (let* ((request-backend 'url-retrieve)
         (server literate-org-rest-server)
         resp)
    (request (concat server "lpy/" path)
        :params params
        :parser 'json-read
        :type type
        :headers '(("Content-Type" . "application/json"))
        :sync t
        :data (encode-coding-string (json-encode data) 'utf-8 t) ; Encode and ensure unibyte
        :complete (cl-function
                   (lambda (&key response &allow-other-keys)
                     (setf literate-org-last-response response)
                     (case (request-response-status-code response)
                       (200 (setf resp (request-response-data response)))

                       (500 (user-error "Literate Python server failed:%s" response))
                       (t (user-error "Failed to request to remote Python server:%s" response))))))
    (let ((type (cdr (assoc 'type resp))))
      (cond ((equal type "error")
             (let ((stdout (cdr (assoc 'stdout resp)))
                   (stderr (cdr (assoc 'stderr resp))))
               (with-current-buffer (get-buffer-create "*literate-org-error*")
                 (erase-buffer)
                 (insert "=== Data ===\n" (encode-coding-string (json-encode data) 'utf-8 t) "\n\n")
                 (insert "==== Error ===\n" (or (cdr (assoc 'error resp)) ""))
                 (when stdout 
                   (insert "\n\n=== stdout ===\n" stdout))
                 (when stderr
                   (insert "\n\n=== stderr ===\n" (cdr (assoc 'stderr resp)))))
               (switch-to-buffer-other-window "*literate-org-error*")))))
    resp))

def stringify_val(member):
    key, val = member
    if isinstance(val, str):
        return key, '"{}"'.format(val)
    if type(val) in (dict, tuple, list):
        return key, _inspect(val)
    return key, f"{str(val)} {str(type(val))}"

def is_trash(member):
    key, val = member
    return (
        key in ["__doc__", "__class__", "__hash__", "__dict__"]
        or ismethod(val)
        or isbuiltin(val)
        or type(val).__name__ == "method-wrapper"
    )

def _pyinspect_inspect_object(obj):
    """
    Turns a **non-primitive** obj into a dictionary of its fields and their values.
    Filters out some built-in magic fields and pretty-prints dictionary values via `json.dumps`.
    Doesn't display methods.
    """
    return dict(stringify_val(m) for m in reversed(getmembers(obj)) if not is_trash(m))

def _pyinspect_add_quotes(key):
    """
    Surrounds string key with extra quotes because Emacs parses them as just symbols
    and makes it hard to distinguish between them and non-string symbols

    >>> _pyinspect_add_quotes("hello")
    '"hello"'

    >>> _pyinspect_add_quotes(1)
    1
    """
    return '"{}"'.format(key) if type(key) is str else key

def trim_seq(seq, elem_cap):

def _pyinspect_take_dict(d: Dict, n: int):
    "Returns a new dictionary with the first n pairs from d"

    def iterator():
        i = 0
        for item in d.items():
            if i == n:
                break
            yield item
            i += 1

    return dict(iterator())

** Introduction

Team workflows still want hand-editable =.py= / =.tsx= / =.vue= /
=.rs= source files — for code review, IDE jump-to-def, or contributors
who don't run Emacs. Import + export round-trips between those files
and org sections so the LP form is not a one-way trap.

** Import
*** How

Import parses source via [[https://tree-sitter.github.io/tree-sitter/using-parsers][tree-sitter]] and emits one org section per
top-level definition. The tree-sitter route avoids per-language
regex/heuristic parsers and keeps name extraction (function /
class / variable) grammar-accurate. A minimal proof:

#+BEGIN_SRC elisp :load no

***** get pieces of code blocks in a source file
To investigate the parse tree, we can use Emacs command [[https://github.com/emacs-mirror/emacs/blob/master/admin/notes/tree-sitter/starter-guide#query-references][treesit-explore-mode]]
#+BEGIN_SRC elisp
(cl-defmethod literate-org-get-code-blocks-in-current-buffer (language module-name section-level)
  (let* ((language (literate-org-alias-language language))
         (tree (literate-org-parse-tree-for-current-file language))
         (root (tsc-root-node tree))
         (count-of-children (tsc-count-children root))
         (module-comment (when (> count-of-children 0)
                           (literate-org-module-comment language root)))
         (current-index 0)
         (next-index 0)
         (next-title nil)
         (code-blocks nil))
    (while (< current-index count-of-children)
      (cl-multiple-value-setq (next-index next-title)
        (literate-org-next-code-block-index language root current-index))
      ;; Absorb a same-line trailing comment into this block so that
      ;; `X = ...  # comment` is not split across two blocks (the
      ;; comment is a separate source_file child in tree-sitter, but
      ;; byte-equivalence requires it to stay on the statement's line).
      ;; A comment on its OWN line (newline between) is NOT absorbed —
      ;; it still flows to the next block as a leading doc comment.
      (let ((end-index next-index)
            (cand (1+ next-index)))
        (when (< cand count-of-children)
          (let ((cand-node (ts-get-nth-child root cand))
                (last-end (tsc-node-end-position (ts-get-nth-child root next-index))))
            (when (and (eq 'comment (ts-node-type cand-node))
                       (not (string-match-p
                             "\n"
                             (buffer-substring-no-properties
                              last-end (tsc-node-start-position cand-node)))))
              (setf end-index cand))))
        (setf code-blocks
                (nconc code-blocks
                       (list
                        (make-literate-org-code-block
                         :title (dired-replace-in-string "\n" " " next-title)
                         :body (let ((begin (tsc-node-start-position (ts-get-nth-child root current-index)))
                                     (end (tsc-node-end-position (ts-get-nth-child root end-index))))
                                 (concat (buffer-substring-no-properties begin end) "\n"))
                         :language language
                         :level section-level))))
        (setf current-index (1+ end-index))))
    (cons module-comment code-blocks)))

The four sections that follow implement the round-trip from a Python
package on disk back into a literate-org document. Each is a single
function:

1. *Prepare a new org section* — emit headers + properties for a

2. *Iterate a directory or file* — top-level entry point that picks

3. *Import a directory* — recursive walk, one section per submodule.
4. *Import a file* — parse a single source file's tree and emit a

#+BEGIN_SRC elisp
(cl-defun literate-org-import (&key (level (or (org-current-level) 0))
                                    module-name module-path)
  "Import source codes from a directory to an org file."
  (interactive "")
  (let* ((module-names (or (ensure-list module-name)
                           (aand (or (org-entry-get (point) "LITERATE_ORG_MODULE" t)
                                     (org-entry-get (point) "LITERATE_ORG_ROOT_MODULE" t))
                                 (split-string it))
                           ;; for all modules in `LITERATE_ORG_ROOT_MODULE'
                           (list nil)))
         (root-path (org-entry-get (point) "LITERATE_ORG_ROOT_MODULE_PATH" t)))
    (dolist (module-name module-names)
      (let ((module-path (or module-path
                             (when root-path
                               (if module-name
                                 (concat root-path "/"
                                         (dired-replace-in-string
                                          "\\." "/" module-name))
                                 root-path)))))
        (if (f-directory? module-path)
          ;; if there are multiple module-names, then we need to create a new org header for each of them.
          ;; it usually happens in global org property `LITERATE_ORG_ROOT_MODULE'.
          (if (> (length module-names) 1)
            (let ((new-level (1+ level)))
              (literate-org-new-org-section-for-a-module
               module-name new-level
               :title module-name
               :path module-path)
              (literate-org-import-directory module-name module-path :level new-level))
            (literate-org-import-directory module-name module-path))
          (literate-org-import-file module-name module-path))
        (message "Finished importing module %s from %s" module-name module-path)))))

(cl-defun literate-org-import-directory (module-name module-directory &key (level (or (org-current-level) 0)))
  (let* ((languages (or (aand (org-entry-get (point) "LITERATE_ORG_LANGUAGES" t)
                              (mapcar #'intern (split-string it)))

**** build org sections for a file

This emits one atomic =#+begin_src= block per top-level definition. When
=literate-org-import-noweb-split= is non-nil (the default) it also stamps
=:LITERATE_ORG_BLOCK_KIND: atomic= on each block and, once the file's
section is built, runs =literate-org--resplit-region= over the freshly
imported subtree — the same engine =literate-org-resplit-buffer= uses — so
an oversized class auto-noweb-splits on import exactly as a manual
=literate-org-resplit-block-at-point= would. Blocks at or below
=literate-org-resplit-threshold= lines stay atomic, so small files import
as before (aside from the property drawer). Set the flag nil to restore
the pre-split output: no drawer, no resplit.

#+BEGIN_SRC elisp
(defcustom literate-org-import-noweb-split t
  "When non-nil, `literate-org-import' stamps each imported block with
`:LITERATE_ORG_BLOCK_KIND: atomic' and resplits oversized ones — a big
class becomes a noweb skeleton plus per-method leaves — over the
just-imported subtree. nil restores the pre-split import output."
  :type 'boolean
  :group 'literate-org)
#+END_SRC

#+BEGIN_SRC elisp
(cl-defun literate-org-import-source-file-to-org
    (language file level module-name &key (with-org-section t))
  (message "literate importing module %s from file %s with language %s" module-name file language)
  (let* ((new-level (1+ level))
         (import-start (point-marker))
         (code-blocks-info
           (with-temp-buffer ;; To avoid running any hooks in original file, which may cause side effects.
             (insert-file-contents file)
             (literate-org-get-code-blocks-in-current-buffer language module-name new-level)))
         (module-comment (car code-blocks-info))
         (module-title (when module-comment
                         (first (split-string module-comment "\n" t nil))))
         (code-blocks (cdr code-blocks-info)))
    (when with-org-section
      (literate-org-new-org-section-for-a-module module-name level
                                                 :language language
                                                 :path file
                                                 :title (or module-title (f-ext module-name) module-name))
      (let* ((module-root-path (org-entry-get (point) "LITERATE_ORG_ROOT_MODULE_PATH" t))
             (root-module (org-entry-get (point) "LITERATE_ORG_ROOT_MODULE" t)))
        (when (and (> (length module-root-path) 0)
                   (not (string-suffix-p "/" module-root-path)))
          (setf module-root-path (concat module-root-path "/")))))

    (cl-loop for block in code-blocks do
             (cl-loop repeat (literate-org-code-block-level block) do (insert "*"))
             (insert " " (literate-org-code-block-title block) "\n")
             (cl-loop for (key . value) in (literate-org-code-block-properties block)
                      do (org-entry-put (point) key value))
             (awhen (literate-org-code-block-body block)
               (insert "#+BEGIN_SRC "
                       (literate-org-babel-language-name (literate-org-code-block-language block))
                       "\n")
               (insert it)
               (insert "\n#+END_SRC\n")
               (when literate-org-import-noweb-split
                 (save-excursion
                   (org-back-to-heading t)
                   (org-entry-put (point) "LITERATE_ORG_BLOCK_KIND" "atomic")))))
    ;; Auto-split oversized atomic blocks (a big class -> noweb skeleton +
    ;; per-method leaves) over the just-imported subtree, reusing the same
    ;; engine as `literate-org-resplit-buffer'.
    (when literate-org-import-noweb-split
      (literate-org--resplit-region import-start (point)))
    (set-marker import-start nil)))
#+END_SRC
**** Resplit an oversized atomic block in place

=literate-org-import= writes one block per top-level definition when it
imports a *fresh* file. A module onboarded the fast way — one atomic
=#+begin_src= block per file — never got that treatment, and re-running
=literate-org-import= would regenerate the whole section and clobber the
prose written since. =literate-org-resplit-block-at-point= is the
in-place variant: it reuses the same tree-sitter parser
(=literate-org-get-code-blocks-in-current-buffer=) to split the single
block into one child per definition, but leaves the heading, properties
drawer, and prose preamble untouched.

It splits only at the top level — exactly what the parser yields. A
section holding a single big class still needs the manual noweb
per-method split (see =lp-noweb-for-big-blocks=), so the command refuses
that case rather than emit a no-op single child.

The split changes whitespace only. Org-Babel trims each block's leading
and trailing blank lines at tangle, so the blank lines that separated the
original top-level definitions collapse; the tangled file is identical to
the original in *content* and differs only in blank-line spacing. If the
project wires a formatter into its tangle path that reproduces its own
style, the spacing comes back; otherwise the tighter spacing is the
accepted result. After resplitting, refresh
=:LITERATE_ORG_CONTAINS_DEFS:= with =lp_metadata_refresh.py=.

***** The split threshold

=literate-org-resplit-buffer= only resplits a block whose body exceeds
this many *lines*. The default targets the genuinely oversized blocks —
the ones that hurt readability — and leaves small atomic blocks alone,
where splitting into sub-headings adds structure without payoff.

#+BEGIN_SRC elisp
(defcustom literate-org-resplit-threshold 120
  "Minimum body *line* count for `literate-org-resplit-buffer' to resplit
a :tangle-bearing atomic block. Blocks at or below this stay atomic —
only oversized blocks are worth the per-definition split."
  :type 'integer
  :group 'literate-org)
#+END_SRC

***** Locating the block and counting its lines

=--first-src-block-in-subtree= finds the section's own =#+begin_src=
without descending into children, so a section that has *already* been
split is left alone. =--section-src-line-count= counts the first src
block's body lines by a plain text scan — no =org-element= — so
=literate-org-resplit-buffer= can size every section in a 25k-line buffer
without the per-heading parse cost that makes the org-element path
quadratic.

#+BEGIN_SRC elisp
(defun literate-org--first-src-block-in-subtree ()
  "Return the org-element of the first src block in the subtree of the
heading at point, or nil. Widens first so a poly-org chunk narrowing
does not hide the heading or the block."
  (save-excursion
    (save-restriction
      (widen)
      (org-back-to-heading t)
      (let ((subtree-end (save-excursion (org-end-of-subtree t t)))
            (found nil))
        (forward-line 1)
        (while (and (not found)
                    (re-search-forward "^[ \t]*#\\+begin_src\\b" subtree-end t))
          (let ((el (org-element-at-point)))
            (if (eq (org-element-type el) 'src-block)
              (setf found el)
              (forward-line 1))))
        found))))

(defun literate-org--section-src-line-count ()
  "Body line count of the first src block in the subtree at point, by a
text scan (no org-element). Returns nil when the section has no src
block. Point may be anywhere at or under the heading."
  (save-excursion
    (save-restriction
      (widen)
      (org-back-to-heading t)
      (let ((end (save-excursion (org-end-of-subtree t t))))
        (when (re-search-forward "^[ \t]*#\\+begin_src" end t)
          (let ((body-start (line-beginning-position 2)))
            (when (re-search-forward "^[ \t]*#\\+end_src" end t)
              (count-lines body-start (line-beginning-position)))))))))
#+END_SRC

***** The noise floor — only split out a chunk over this many lines

Replaces the per-*file* line threshold with a per-*definition* noise
floor. A top-level definition (or a class method) earns its own section
only when its body exceeds this many lines; smaller definitions group
with their neighbours so the split stays readable. Structure decides
*whether* to split; this number only decides *what is too small to be
worth isolating*.

#+BEGIN_SRC elisp
(defcustom literate-org-noweb-min-chunk-lines 10
  "A definition must exceed this many *body lines* to earn its own
section / noweb chunk. Smaller definitions are grouped with neighbours so
the split does not fragment into one-liner sections."
  :type 'integer
  :group 'literate-org)
#+END_SRC

***** Tree helpers shared by every language

Small wrappers over tree-sitter that the language-specific
=literate-org-split-node= methods build on. =--dedent= strips a fixed
indent from each line so a method extracted from inside a class can be
stored at column 0 — Org-Babel re-indents it to the =<<chunk>>=
placeholder's column on tangle, so no manual re-indent is needed.

#+BEGIN_SRC elisp
(defun literate-org--node-line-count (node)
  "Line span of tree-sitter NODE."
  (count-lines (tsc-node-start-position node) (tsc-node-end-position node)))

(defun literate-org--dedent (text n)
  "Remove up to N leading spaces from each line of TEXT."
  (if (<= n 0) text
    (replace-regexp-in-string (concat "^" (make-string n ?\s)) "" text)))

(defun literate-org--column-at (pos)
  "Visual column of buffer position POS."
  (save-excursion (goto-char pos) (current-column)))

(defun literate-org--child-of-type (node type)
  "First child of NODE whose node-type is TYPE, or nil."
  (cl-loop for i below (tsc-count-children node)
           for c = (tsc-get-nth-child node i)
           when (eq type (tsc-node-type c)) return c))

(defun literate-org--brace-bounds (body)
  "For a braced body container BODY, return (INTERIOR-START . INTERIOR-END)
spanning between the opening and closing brace tokens."
  (let (op cl)
    (cl-loop for i below (tsc-count-children body)
             for c = (tsc-get-nth-child body i)
             do (let ((ty (tsc-node-type c)))
                  (cond ((equal ty "{") (setq op c))
                        ((equal ty "}") (setq cl c)))))
    (cons (tsc-node-end-position op) (tsc-node-start-position cl))))

(defun literate-org--members-of-types (body types)
  "Direct children of BODY whose node-type is in TYPES."
  (cl-loop for i below (tsc-count-children body)
           for c = (tsc-get-nth-child body i)
           when (memq (tsc-node-type c) types) collect c))

(defun literate-org--big-members (members)
  "From MEMBERS keep those over the chunk floor, as (START END TITLE)."
  (cl-loop for m in members
           when (> (literate-org--node-line-count m) literate-org-noweb-min-chunk-lines)
           collect (list (tsc-node-start-position m) (tsc-node-end-position m)
                         (let ((nm (tsc-get-child-by-field m :name)))
                           (if nm (tsc-node-text nm) "def")))))
#+END_SRC

***** =literate-org-split-node= — the per-language container rule

Given a tree-sitter node, return a plist describing how to noweb-split it
into a skeleton + chunks, or nil to leave it as one atomic block. A
container splits only when it holds *≥2 members over the noise floor* —
that is the structure-driven decision that replaces a line constant. Each
language method answers the same two questions in its own grammar's
terms: what is the body container (=block= / =declaration_list= /
=class_body=), and what are its member definitions. Adding a language is
one =cl-defmethod=, no edits elsewhere.

#+BEGIN_SRC elisp
(cl-defgeneric literate-org-split-node (language node-type node)
  "Return a split plist (:ref :envelope :indent :istart :iend :big) for
NODE, or nil to keep it atomic."
  (:method (_l _t _n) nil))

(defun literate-org--split-braced (node body members-types name lang-indent-default)
  "Shared split for brace-delimited bodies (rust impl, ts class). NODE's
body container is BODY; its members are children of types MEMBERS-TYPES;
NAME names the chunk."
  (let* ((bb (literate-org--brace-bounds body))
         (members (literate-org--members-of-types body members-types))
         (big (literate-org--big-members members))
         (indent (if members (literate-org--column-at (tsc-node-start-position (car members)))
                   lang-indent-default))
         (sig (buffer-substring-no-properties (tsc-node-start-position node)
                                              (tsc-node-start-position body))))
    (when (>= (length big) 2)
      (list :ref (concat name "-body") :indent indent
            :envelope (concat sig "{\n" (make-string indent ?\s) "<<" name "-body>>\n}")
            :istart (car bb) :iend (cdr bb) :big big))))

;; Python class_definition and function_definition share a block-based
;; body and split the same way — on the block's nested (function |
;; decorated)_definition members. So a big function with >=2 big nested
;; helpers noweb-splits exactly like a class with >=2 big methods.
(defun literate-org--python-block-split (node)
  "Split a python class_definition / function_definition by the nested
definitions in its block, or nil if fewer than two are over the floor."
  (let ((block (literate-org--child-of-type node 'block)))
    (when block
      (let* ((indent (literate-org--column-at (tsc-node-start-position block)))
             (members (literate-org--members-of-types
                       block '(function_definition decorated_definition)))
             (big (literate-org--big-members members))
             (name (let ((nm (tsc-get-child-by-field node :name)))
                     (if nm (tsc-node-text nm) "block")))
             (sig (buffer-substring-no-properties (tsc-node-start-position node)
                                                  (tsc-node-start-position block))))
        (when (>= (length big) 2)
          (list :ref (concat name "-body") :indent indent
                :envelope (concat sig "<<" name "-body>>")
                :istart (tsc-node-start-position block)
                :iend (tsc-node-end-position block) :big big))))))

(cl-defmethod literate-org-split-node ((_l (eql python)) (_t (eql class_definition)) node)
  (literate-org--python-block-split node))

(cl-defmethod literate-org-split-node ((_l (eql python)) (_t (eql function_definition)) node)
  (literate-org--python-block-split node))

(cl-defmethod literate-org-split-node ((_l (eql rust)) (_t (eql impl_item)) node)
  (awhen (literate-org--child-of-type node 'declaration_list)
    (literate-org--split-braced
     node it '(function_item)
     (let ((ty (literate-org--child-of-type node 'type_identifier)))
       (if ty (tsc-node-text ty) "Impl"))
     4)))

(cl-defmethod literate-org-split-node ((_l (eql rust)) (_t (eql function_item)) node)
  (awhen (literate-org--child-of-type node 'block)
    (literate-org--split-braced
     node it '(function_item)
     (let ((nm (tsc-get-child-by-field node :name)))
       (if nm (tsc-node-text nm) "fn"))
     4)))

(cl-defmethod literate-org-split-node ((_l (eql typescript)) (_t (eql class_declaration)) node)
  (awhen (literate-org--child-of-type node 'class_body)
    (literate-org--split-braced
     node it '(method_definition public_field_definition)
     (let ((ty (literate-org--child-of-type node 'type_identifier)))
       (if ty (tsc-node-text ty) "Class"))
     2)))

(cl-defmethod literate-org-split-node ((_l (eql typescript)) (_t (eql function_declaration)) node)
  (awhen (literate-org--child-of-type node 'statement_block)
    (literate-org--split-braced
     node it '(function_declaration)
     (let ((nm (tsc-get-child-by-field node :name)))
       (if nm (tsc-node-text nm) "fn"))
     2)))

***** Tiling a container body into leaf chunks

The interior of a container is *tiled* by its big members: each big
member becomes its own leaf; the text between big members (small methods,
fields, the docstring, punctuation) accumulates into "members" leaves.
Tiling by text slice — not by reconstructing from member nodes —
preserves every byte between definitions (TypeScript field semicolons,
blank lines) that a node-only rebuild would drop. Each leaf is dedented
so Org-Babel can re-indent it under the skeleton placeholder.

#+BEGIN_SRC elisp
(defun literate-org--tile-leaves (split)
  "Tile the container interior described by SPLIT into a list of
(TITLE . BODY) leaves, big members isolated and small runs grouped."
  (let* ((indent (plist-get split :indent))
         (istart (plist-get split :istart))
         (iend (plist-get split :iend))
         (big (sort (copy-sequence (plist-get split :big))
                    (lambda (a b) (< (car a) (car b)))))
         (cursor istart) (leaves nil))
    (dolist (b big)
      (cl-destructuring-bind (ms me ttl) b
        (when (> ms cursor)
          (let ((seg (literate-org--dedent
                      (buffer-substring-no-properties cursor ms) indent)))
            (unless (string-blank-p seg) (push (cons "members" seg) leaves))))
        (push (cons ttl (literate-org--dedent
                         (buffer-substring-no-properties ms me) indent))
              leaves)
        (setq cursor me)))
    (when (< cursor iend)
      (let ((seg (literate-org--dedent
                  (buffer-substring-no-properties cursor iend) indent)))
        (unless (string-blank-p seg) (push (cons "members" seg) leaves))))
    (nreverse leaves)))
#+END_SRC

***** From a file body to a list of units

=--file-units= tiles the *whole file body* the same way: top-level
definitions over the floor become their own units; smaller top-level
statements (imports, module constants, the =app = Typer()= cluster) group
into "members" units instead of fragmenting into one-liner sections. A
big top-level class that =literate-org-split-node= accepts becomes a
=noweb= unit; everything else is =atomic=. Noweb is suppressed when the
section sits too deep for its leaves to stay within the depth-5 cap.

#+BEGIN_SRC elisp
(defun literate-org--unit-title (language node)
  "A heading title for top-level NODE."
  (condition-case nil
      (literate-org-node-name language (tsc-node-type node) node)
    (error (let ((nm (tsc-get-child-by-field node :name)))
             (if nm (tsc-node-text nm) (symbol-name (tsc-node-type node)))))))

(defun literate-org--file-units (body language level)
  "Tile BODY into a list of units. Each unit is either
(atomic TITLE BODY) or (noweb TITLE REF ENVELOPE LEAVES)."
  (with-temp-buffer
    (insert body)
    (let* ((language (literate-org-alias-language language))
           (tree (literate-org-parse-tree-for-current-file language))
           (root (tsc-root-node tree))
           (count (tsc-count-children root))
           (floor literate-org-noweb-min-chunk-lines)
           (allow-noweb (<= (+ level 2) 5))
           (cursor (point-min)) (units nil))
      (cl-loop for i below count
               for node = (tsc-get-nth-child root i) do
               (when (> (literate-org--node-line-count node) floor)
                 (let ((ns (tsc-node-start-position node))
                       (ne (tsc-node-end-position node)))
                   (when (> ns cursor)
                     (let ((seg (buffer-substring-no-properties cursor ns)))
                       (unless (string-blank-p seg)
                         (push (list 'atomic "members" seg) units))))
                   (let ((split (and allow-noweb
                                     (literate-org-split-node
                                      language (tsc-node-type node) node))))
                     (if split
                         (push (list 'noweb (literate-org--unit-title language node)
                                     (plist-get split :ref) (plist-get split :envelope)
                                     (literate-org--tile-leaves split))
                               units)
                       (push (list 'atomic (literate-org--unit-title language node)
                                   (buffer-substring-no-properties ns ne))
                             units)))
                   (setq cursor ne))))
      (when (< cursor (point-max))
        (let ((seg (buffer-substring-no-properties cursor (point-max))))
          (unless (string-blank-p seg)
            (push (list 'atomic "members" seg) units))))
      (nreverse units))))
#+END_SRC

***** Emitting the units

Walks the unit list and writes Org. An =atomic= unit is a sub-heading
with a bare =:padline no= src block that inherits the file section's
=:tangle=. A =noweb= unit is a skeleton sub-heading (its own
=:header-args: :tangle no :noweb-ref REF= so the leaves inherit it, plus
=:CUSTOM_ID:= + =:LITERATE_ORG_BLOCK_KIND: skeleton:=) whose one block
tangles the envelope with =:noweb yes=, followed by one =noweb-leaf=
sub-sub-heading per chunk. Bodies are re-escaped so a column-0 =*= /
=#+= cannot terminate the block, and stamped =:LITERATE_ORG_CONTAINS_DEFS:=
is left to =lp_metadata_refresh.py=.

#+BEGIN_SRC elisp
(defun literate-org--anchor (ref)
  "A CUSTOM_ID slug derived from a noweb REF."
  (downcase (replace-regexp-in-string "[^A-Za-z0-9]+" "-" ref)))

(defun literate-org--emit-src (lang body header-extra)
  "Insert a src block of LANG with BODY; HEADER-EXTRA is the header
remainder after the language (e.g. \" :padline no\")."
  (insert "#+BEGIN_SRC " lang header-extra "\n")
  (insert (org-escape-code-in-string body))
  (unless (string-suffix-p "\n" body) (insert "\n"))
  (insert "#+END_SRC\n\n"))

(defun literate-org--emit-units (units language tangle level)
  "Emit UNITS as Org under a file section at LEVEL tangling to TANGLE."
  (let ((lang (literate-org-babel-language-name language))
        (clevel (1+ level)))
    (dolist (u units)
      (pcase (car u)
        ('atomic
         (cl-destructuring-bind (_ title body) u
           (insert (make-string clevel ?*) " " title "\n")
           (insert "Prose pending (auto-split): " title ".\n\n")
           (literate-org--emit-src lang body " :padline no")))
        ('noweb
         (cl-destructuring-bind (_ title ref envelope leaves) u
           ;; CUSTOM_ID must NOT equal the noweb-ref: Org resolves
           ;; =<<Foo-body>>= against CUSTOM_ID/NAME targets too (case-
           ;; insensitively), so a CUSTOM_ID "foo-body" hijacks the
           ;; reference and expands it to the whole subtree. Derive the
           ;; anchor from the class name (ref minus the "-body" suffix).
           (let ((cid (literate-org--anchor
                       (replace-regexp-in-string "-body\\'" "" ref))))
             (insert (make-string clevel ?*) " " title "\n")
             (insert ":PROPERTIES:\n:CUSTOM_ID: " cid
                     "\n:LITERATE_ORG_BLOCK_KIND: skeleton\n:header-args: :tangle no :noweb-ref "
                     ref "\n:END:\n")
             (insert "Prose pending (auto-split): =" title "= skeleton.\n\n")
             (literate-org--emit-src lang envelope
                                     (concat " :tangle " tangle " :noweb yes :noweb-ref \"\""))
             ;; Leaves are collected into the chunk by :noweb-ref, never
             ;; tangled directly, so they need no :padline.
             (dolist (lf leaves)
               (insert (make-string (1+ clevel) ?*) " " (car lf) "\n")
               (insert ":PROPERTIES:\n:LITERATE_ORG_BLOCK_KIND: noweb-leaf\n:LITERATE_ORG_NOWEB_PARENT: "
                       cid "\n:END:\n")
               (insert "Prose pending (auto-split): " (car lf) ".\n\n")
               (literate-org--emit-src lang (cdr lf) "")))))))))
#+END_SRC

***** The resplit command

The core does the work and returns the unit count; the interactive
command wraps it in =save-excursion= and reports the verification step.
Every guard runs *before* the first buffer edit, so a refused section is
left exactly as it was. A single big class becomes one =noweb= unit (the
split is the point); a section with nothing worth isolating raises
=user-error=.

#+BEGIN_SRC elisp
(defun literate-org--resplit-section ()
  "Split the atomic :tangle block in the section at point into per-unit
sub-headings (top-level definitions atomic; big classes noweb). Returns
the unit count. Signals `user-error' when the section has no :tangle
target, sits at the depth-5 cap, has no src block, or has nothing worth
splitting."
  (widen)
  (org-back-to-heading t)
  (let* ((level (org-current-level))
         (header-args (org-entry-get (point) "header-args" t))
         (tangle (and header-args
                      (string-match ":tangle +\\(\\S-+\\)" header-args)
                      (match-string 1 header-args)))
         (heading (point-marker)))
    (unless (and tangle (not (string= tangle "no")))
      (user-error "No :tangle target — nothing to resplit"))
    (when (>= level 5)
      (user-error "Section at depth %d — children would breach the depth-5 cap" level))
    (let ((el (literate-org--first-src-block-in-subtree)))
      (unless el
        (user-error "No src block in this section"))
      (let* ((lang (intern (or (org-element-property :language el) "python")))
             (body (org-element-property :value el))
             (units (literate-org--file-units body lang level))
             (beg (org-element-property :begin el))
             (end (org-element-property :end el)))
        (when (or (null units)
                  (and (= (length units) 1) (eq (caar units) 'atomic)))
          (user-error "Nothing worth splitting in this section"))
        ;; Inhibit modification hooks across the bulk edit: in a poly-org
        ;; buffer every insert otherwise fires polymode's after-change
        ;; rescan over the whole file (O(file) per insert — quadratic, and
        ;; its chunk-matcher rx chokes on the freshly inserted text). The
        ;; saved file is correct; refontify lazily afterwards.
        (let ((inhibit-modification-hooks t))
          (goto-char beg)
          (delete-region beg end)
          (literate-org--emit-units units lang tangle level)
          (goto-char heading)
          (when (org-entry-get (point) "LITERATE_ORG_BLOCK_KIND")
            (org-entry-delete (point) "LITERATE_ORG_BLOCK_KIND")))
        (length units)))))

(defun literate-org-resplit-block-at-point ()
  "Split the atomic :tangle block in the section at point into per-unit
sub-headings, noweb-splitting big classes, preserving the prose preamble.

Changes whitespace only at tangle (Org-Babel trims block boundaries);
content is preserved. Verify with the project's tangle + `git diff' gate,
then refresh metadata with `lp_metadata_refresh.py'. In a poly-org buffer
the work runs in the org-mode base buffer."
  (interactive)
  (let ((base (or (buffer-base-buffer) (current-buffer)))
        (pos (point)))
    (with-current-buffer base
      (save-excursion
        (goto-char pos)
        (let ((n (literate-org--resplit-section)))
          (message "Resplit into %d units. Verify: tangle + `git diff' content-clean, then refresh metadata."
                   n))))))
#+END_SRC

***** Batch resplit across a region

Collects candidate headings by a *plain text scan* for the
=:LITERATE_ORG_BLOCK_KIND: atomic= drawer line — deliberately NOT
=org-map-entries=, which over a 25k-line poly-org buffer is both slow
(O(headings) =org-end-of-subtree= calls) and prone to "Invalid search
bound" when a callback widens mid-iteration. The scan records a marker
per oversized atomic section, then resplits them *bottom-up* so each
edit only shifts text below the markers still to process. Sections with
nothing worth splitting raise =user-error= and are tallied.

Parameterising the scan over =[BEG, END]= is what lets one core serve two
scopes: =literate-org-resplit-buffer= hands it the whole buffer, while a
fresh =literate-org-import= hands it only the subtree it just emitted — so
import splits its own oversized classes without disturbing sections it did
not write. Returns =(DONE . SKIPPED-HEADINGS)=.

#+BEGIN_SRC elisp
(defun literate-org--resplit-region (beg end)
  "Resplit every :tangle-bearing atomic block between BEG and END whose
body exceeds `literate-org-resplit-threshold' lines. Locates candidates by
a text scan for the `:LITERATE_ORG_BLOCK_KIND: atomic' drawer line, then
resplits them bottom-up so each edit only shifts text below the markers
still to process. Returns (DONE . SKIPPED-HEADINGS).

Wrapped in `save-restriction' because `literate-org--resplit-section'
widens unconditionally; without it a caller inside a narrowed poly-org
chunk (e.g. import run from a poly-org base) would silently lose its
narrowing."
  (let ((positions nil) (done 0) (skipped nil))
    (save-restriction
      (save-excursion
        (goto-char beg)
        (while (re-search-forward "^:LITERATE_ORG_BLOCK_KIND: atomic[ \t]*$" end t)
          (save-excursion
            (org-back-to-heading t)
            (awhen (literate-org--section-src-line-count)
              (when (> it literate-org-resplit-threshold)
                (push (copy-marker (point)) positions)))))
        (dolist (m (sort positions (lambda (a b) (> (marker-position a) (marker-position b)))))
          (goto-char m)
          (condition-case _
              (progn (literate-org--resplit-section) (cl-incf done))
            (user-error (push (org-get-heading t t t t) skipped))))))
    (cons done skipped)))
#+END_SRC

=literate-org-resplit-buffer= widens and runs the region core over the
whole buffer.

#+BEGIN_SRC elisp
(defun literate-org-resplit-buffer ()
  "Resplit every :tangle-bearing atomic block in the current buffer whose
body exceeds `literate-org-resplit-threshold' lines. Runs in the org-mode
base buffer; collects via text scan to stay fast on big poly-org buffers."
  (interactive)
  (with-current-buffer (or (buffer-base-buffer) (current-buffer))
    (save-excursion
      (save-restriction
        (widen)
        (let ((r (literate-org--resplit-region (point-min) (point-max))))
          (message "literate-org-resplit-buffer: split %d sections; %d skipped (%s)"
                   (car r) (length (cdr r)) (cdr r)))))))
#+END_SRC

***** Tests — every language round-trips through noweb

runs =--file-units= + =--emit-units= into a temp Org buffer, tangles it,
and asserts the tangled output equals the original *content* (non-blank
lines) — proving the noweb skeleton + leaves + Org-Babel's auto-indent

(defun literate-org--roundtrip-content-eq (lang src)
  "Resplit SRC for LANG and tangle it; return non-nil when the tangled
output matches SRC's non-blank content."
  (let* ((path (make-temp-file "lo-rt" nil
                               (concat "." (literate-org-file-extension lang))))
         (units (literate-org--file-units src lang 1))
         (orgf (make-temp-file "lo-rt" nil ".org")))
    (with-temp-buffer
      (insert "#+PROPERTY: header-args :tangle no\n* f\n:PROPERTIES:\n:header-args: :tangle "
              path "\n:END:\nprose.\n")
      (literate-org--emit-units units lang path 1)
      (write-region (point-min) (point-max) orgf))
    (delete-file path)
    (let ((org-confirm-babel-evaluate nil)) (org-babel-tangle-file orgf))
    (cl-flet ((nb (s) (seq-remove #'string-empty-p
                                  (mapcar #'string-trim (split-string s "\n")))))
      (equal (nb src)
             (nb (with-temp-buffer (insert-file-contents path) (buffer-string)))))))

***** How to get the name of a parse node

The Python dispatch table for =literate-org-node-name=. Each
=cl-defmethod= overrides one tree-sitter node type to a human-readable
section title. The methods are siblings in a dispatch table, so they
live in one src block here rather than spread across separate
sub-sections — that keeps the table scannable and avoids depth-6
nesting that an LLM would otherwise drill into without zooming out.

Two name-bearing nodes plus six trivial constants fold into two
=defconst= alists consulted by one catch-all =cl-defmethod= (same
shape used by the Rust and TSX sections). The five nodes with
per-shape logic keep their own specialised methods:
=decorated_definition= chains its decorators; =assignment= /
=augmented_assignment= / =call= truncate the rendered source via
=literate-org-max-title-size= and fall back to the =:left= /
=:function= field when over the cap; =expression_statement= unwraps
its first child.

#+BEGIN_SRC elisp
(defconst literate-org-python--name-labels
  '((function_definition . "Function")
    (class_definition    . "Class"))
  "Python node types whose section title is \"<label> <:name>\".")

(defconst literate-org-python--constant-labels
  '((try_statement         . "Try statement")
    (while_statement       . "While statement")
    (string                . "String")
    (comment               . "Comment")
    (import                . "Import")
    (import_from_statement . "Import"))
  "Python node types whose section title is a fixed string.")

;; Catch-all: alist lookup; on miss, fall through to the generic default.
(cl-defmethod literate-org-node-name ((_language (eql python)) node-type node)
  (cond
    ((alist-get node-type literate-org-python--name-labels)
     (format "%s %s"
             (alist-get node-type literate-org-python--name-labels)
             (tsc-node-text (tsc-get-child-by-field node :name))))
    ((alist-get node-type literate-org-python--constant-labels))
    (t (cl-call-next-method))))

;; decorated_definition — chain of @decorators followed by the underlying definition.
(cl-defmethod literate-org-node-name ((language (eql python))
                                      (_node-type (eql decorated_definition)) node)
  (let* ((name-list nil)
         (index 0)
         (child (tsc-get-nth-child node index)))
    (while (eq 'decorator (tsc-node-type child))
      (setf name-list (nconc name-list (list (tsc-node-text (tsc-get-nth-child child 1)))))
      (incf index)
      (setf child (tsc-get-nth-child node index)))
    (format "@%s: %s" (mapconcat 'identity name-list " ")
            (literate-org-node-name language (tsc-node-type child) child))))

;; assignment / augmented_assignment — source text, or :left child when too long.
;; Two cl-defmethods because dispatch is on node-type, but body is identical.
(cl-defmethod literate-org-node-name ((_language (eql python))
                                      (_node-type (eql assignment)) node)
  (let ((code (dired-replace-in-string "\n" " " (tsc-node-text node))))
    (if (<= (length code) literate-org-max-title-size)
      (format "Assignment %s" code)
      (format "Assignment %s" (tsc-node-text (tsc-get-child-by-field node :left))))))

(cl-defmethod literate-org-node-name ((_language (eql python))
                                      (_node-type (eql augmented_assignment)) node)
  (let ((code (dired-replace-in-string "\n" " " (tsc-node-text node))))
    (if (<= (length code) literate-org-max-title-size)
      (format "Assignment %s" code)
      (format "Assignment %s" (tsc-node-text (tsc-get-child-by-field node :left))))))

;; call — source text, or :function child when too long.
(cl-defmethod literate-org-node-name ((_language (eql python))
                                      (_node-type (eql call)) node)
  (let ((call-code (dired-replace-in-string "\n" " " (tsc-node-text node))))
    (if (<= (length call-code) literate-org-max-title-size)
      (format "Call %s" call-code)
      (format "Call %s" (tsc-node-text (tsc-get-child-by-field node :function))))))

;; expression_statement — recurse into the wrapped expression.
(cl-defmethod literate-org-node-name ((language (eql python))
                                      (_node-type (eql expression_statement)) node)
  (let ((new-node (tsc-get-nth-child node 0)))
    (literate-org-node-name language (ts-node-type new-node) new-node)))

***** get the next code block index

The Python and TSX scanners share the same shape — walk the
=source_file= children, classify each node as one of:

  - *import-like* (Python =import_statement= / =import_from_statement=,
    TSX =import_statement=); collapse contiguous runs (including
    glue comments) into one "Import statements" block.

  - *definition* (Python class / function / decorated_definition;
    TSX type / enum / interface / lexical_declaration / variable_
    declarator / export_statement / function_declaration); emit
    one block per definition titled via =literate-org-node-name=.

  - *expression_statement*: unwrap (Python: docstring / assignment
    with optional docstring follow-up / fallback; TSX: same path
    works because the case shape is grammar-agnostic).

  - *trailing comment* on the last child of the source file: emit
    as its own block when nothing follows.

The Rust scanner uses a different classification (item / use-like /
prelude with attribute + doc handling) and stays as its own
=cl-defmethod=. The shared helper accepts the language-specific
import-type and definition-type sets as parameters so Python and
TSX collapse to thin wrappers.

#+BEGIN_SRC elisp
(defun literate-org--scan-flat-with-import-group
    (language root start-index import-types def-types)
  "Walk ROOT's children from START-INDEX classifying nodes for the
flat import-then-defs grammar pattern (Python, TSX).

IMPORT-TYPES is a list of node-type symbols treated as import-like.
DEF-TYPES is a list of node-type symbols treated as definitions —
each emits its own block titled via `literate-org-node-name'.

Returns =(NEXT-INDEX . TITLE)= as a `values' tuple, matching the
generic contract of `literate-org-next-code-block-index'."
  (cl-block nil
    (let* ((max-index (1- (tsc-count-children root)))
           (current-index start-index)
           (current-node nil)
           (current-node-type nil)
           (next-index nil)
           (next-node-type nil)
           (next-node nil)
           (collected-valid-node-types nil)
           (previous-node nil)
           (previous-node-type nil))
      (while (<= current-index max-index)
        (setf previous-node current-node)
        (setf previous-node-type current-node-type)
        (setf current-node (ts-get-nth-child root current-index))
        (setf current-node-type (ts-node-type current-node))
        (unless (eq 'comment current-node-type)
          (push current-node-type collected-valid-node-types))

        (when (= current-index max-index)
          (return (values max-index
                          (literate-org-node-name
                           language current-node-type current-node))))
        (setf next-index (1+ current-index)
              next-node (ts-get-nth-child root next-index)
              next-node-type (ts-node-type next-node))
        (cond
          ;; Trailing comment after a non-import — close the previous
          ;; def's block at the comment boundary.
          ((eq current-node-type 'comment)
           (if (and previous-node-type
                    (not (eq previous-node-type 'comment))
                    (memq (first collected-valid-node-types) import-types)
                    (not (or (memq next-node-type import-types)
                             (eq next-node-type 'comment))))
             (return (values (1- current-index)
                             (literate-org-node-name
                              language previous-node-type previous-node)))
             (incf current-index)))
          ;; Import-like run — collapse contiguous imports (with glue comments).
          ((memq current-node-type import-types)
           (cond ((or (memq next-node-type import-types)
                      (eq next-node-type 'comment))
                  (incf current-index))
                 (t
                  (return (values current-index "Import statements")))))
          ;; Definition — emit its title and stop.
          ((memq current-node-type def-types)
           (return (values current-index
                           (literate-org-node-name
                            language current-node-type current-node))))
          ;; expression_statement — unwrap one level, dispatch on inner.
          ((eq current-node-type 'expression_statement)
           (let* ((first-child-node (ts-get-nth-child current-node 0))
                  (first-child-node-type (ts-node-type first-child-node)))
             (cl-case first-child-node-type
               (comment
                (incf next-index)
                (setf next-node (ts-get-nth-child root next-index))
                (setf next-node-type (ts-node-type next-node)))
               (string
                (return (values current-index "Docstring")))
               (assignment
                (if (and (eq 'expression_statement next-node-type)
                         (eq 'string (ts-node-type (ts-get-nth-child next-node 0))))
                  (incf current-index)
                  (return (values current-index
                                  (literate-org-node-name
                                   language first-child-node-type first-child-node)))))
               (t
                (return (values current-index
                                (literate-org-node-name
                                 language first-child-node-type first-child-node)))))))
          (t
           (incf current-index)))))))

(cl-defmethod literate-org-next-code-block-index ((language (eql python))
                                                  root start-index)
  (literate-org--scan-flat-with-import-group
   language root start-index
   '(import_statement import_from_statement)
   '(decorated_definition class_definition function_definition)))
#+END_SRC

**** Rust

Rust's tree-sitter grammar is clean: every top-level item
(=function_item=, =struct_item=, =enum_item=, =trait_item=, =union_item=,
=mod_item=, =const_item=, =static_item=, =type_item=, =macro_definition=,
=impl_item=, =foreign_mod_item=) is a direct child of =source_file=, and
most carry a =:name= field. The dispatch mirrors Python's; only a few
Rust-specific shapes need hand-coding:

- Outer attributes (=#[derive(...)]=) and doc comments (=///=) precede
  the item they decorate, so =next-code-block-index= slides past them
  and the item's heading absorbs them.
- Contiguous =use= / =extern crate= declarations fold into one "Use
  declarations" block, the same way Python collapses =import_statement=
  runs.
- Inner attributes (=#![...]=) at the file top must be skipped before
  scanning for the =//!= module doc — otherwise a crate that opens with
  =#![allow(dead_code)]= loses its module comment.
- Top-level macro invocations come in two shapes: =lazy_static! { ... }=
  parses as a direct =macro_invocation=, while =include_str!(...);=
  wraps one in an =expression_statement=. Both emit "Macro call <name>"
  via the =:macro= field.

***** file extension and module comment

Extension =.rs=. The module comment is the contiguous run of =//!= lines
at the top, *after* any leading =inner_attribute_item= /
=attribute_item= nodes — the gap that bit the prototype on crates
opening with =#![allow(...)]=.

#+BEGIN_SRC elisp
(cl-defmethod literate-org-file-extension ((_language (eql rust)))
  "rs")

(cl-defmethod literate-org-module-comment ((language (eql rust)) node)
  (when (eq 'source_file (tsc-node-type node))
    (let* ((index 0)
           (count (tsc-count-children node))
           (child nil))
      ;; Skip past leading #![...] / #[...] before scanning for //! lines.
      (while (and (< index count)
                  (memq (tsc-node-type
                         (setq child (tsc-get-nth-child node index)))
                        '(inner_attribute_item attribute_item)))
        (cl-incf index))
      (let ((parts nil))
        (while (and (< index count)
                    (eq 'line_comment
                        (tsc-node-type
                         (setq child (tsc-get-nth-child node index))))
                    (string-prefix-p "//!" (tsc-node-text child)))
          (push (string-trim-left (substring (tsc-node-text child) 3))
                parts)
          (cl-incf index))
        (when parts
          (mapconcat #'identity (nreverse parts) "\n"))))))
#+END_SRC

***** How to get the name of a parse node

Most of Rust's top-level items are name-bearing (function / struct /
enum / trait / union / const / static / type / mod / macro_definition)
and need exactly the same dispatch shape: =(format "<Label> %s"
(tsc-node-text (tsc-get-child-by-field node :name)))=. The remaining
nodes either render as a fixed string (use, extern crate, attribute,
comment, …) or carry per-shape logic (=impl_item= renders
=impl <Trait> for <Type>=; =macro_invocation= reads =:macro=;
=expression_statement= unwraps).

We keep two alists — =--name-labels= for the :name-pulling node
types, =--constant-labels= for the trivial fixed-string ones — and
one catch-all =cl-defmethod (eql rust)= that consults them. The
per-shape methods (impl / macro_invocation / expression_statement)
keep their own =cl-defmethod=s and shadow the catch-all because
specifying node-type as =(eql X)= is more specific than =t=.

#+BEGIN_SRC elisp
(defconst literate-org-rust--name-labels
  '((function_item    . "Function")
    (struct_item      . "Struct")
    (enum_item        . "Enum")
    (trait_item       . "Trait")
    (union_item       . "Union")
    (const_item       . "Const")
    (static_item      . "Static")
    (type_item        . "Type")
    (mod_item         . "Mod")
    (macro_definition . "Macro"))
  "Rust node types whose section title is \"<label> <:name>\".
One alist row replaces what used to be ten 3-line `cl-defmethod's.")

(defconst literate-org-rust--constant-labels
  '((use_declaration          . "Use")
    (extern_crate_declaration . "Extern crate")
    (foreign_mod_item         . "Extern block")
    (attribute_item           . "Attribute")
    (inner_attribute_item     . "Inner attribute")
    (line_comment             . "Comment")
    (block_comment            . "Comment"))
  "Rust node types whose section title is a fixed string irrespective
of node content.")

;; Catch-all: any rust node-type that isn't shadowed by a more
;; specific cl-defmethod is dispatched by alist lookup.
(cl-defmethod literate-org-node-name ((_language (eql rust)) node-type node)
  (cond
    ((alist-get node-type literate-org-rust--name-labels)
     (format "%s %s"
             (alist-get node-type literate-org-rust--name-labels)
             (tsc-node-text (tsc-get-child-by-field node :name))))
    ((alist-get node-type literate-org-rust--constant-labels))
    (t (cl-call-next-method))))

;; impl_item — :type required, :trait optional. Not in the alist
;; because the result depends on which fields are present.
(cl-defmethod literate-org-node-name ((_language (eql rust))
                                      (_node-type (eql impl_item)) node)
  (let* ((trait-node (ignore-errors (tsc-get-child-by-field node :trait)))
         (type-node  (ignore-errors (tsc-get-child-by-field node :type)))
         (trait-text (and trait-node (tsc-node-text trait-node)))
         (type-text  (and type-node  (tsc-node-text type-node))))
    (if trait-text
      (format "Impl %s for %s" trait-text (or type-text "?"))
      (format "Impl %s" (or type-text "?")))))

;; macro_invocation — :macro field gives the macro path (e.g. "lazy_static",
;; "include_str", "my_crate::register"); fall back to the whole node text.
(cl-defmethod literate-org-node-name ((_language (eql rust))
                                      (_node-type (eql macro_invocation)) node)
  (let ((macro-node (ignore-errors (tsc-get-child-by-field node :macro))))
    (format "Macro call %s" (tsc-node-text (or macro-node node)))))

;; expression_statement — recurse into the wrapped expression.
(cl-defmethod literate-org-node-name ((language (eql rust))
                                      (_node-type (eql expression_statement)) node)
  (let ((inner (tsc-get-nth-child node 0)))
    (literate-org-node-name language (tsc-node-type inner) inner)))

***** get the next code block index

Three node sets drive the scan: *items* (own block), *use-like*
declarations (folded into one "Use declarations" block), and *preludes*
(attributes / comments, slide forward into the next item). An
=expression_statement= wrapping a =macro_invocation= also counts as an
item — that's how =include_str!(...);= gets its own block instead of
falling through.

#+BEGIN_SRC elisp
;; defconst rather than defvar — these are semantic constants over the
;; tree-sitter-rust grammar, and we want a reload of this org file to
;; refresh them even if a prototype with the same name was eval'd earlier.
(defconst literate-org-rust--item-types
  '(function_item struct_item enum_item trait_item union_item
    mod_item const_item static_item type_item macro_definition
    impl_item foreign_mod_item macro_invocation)
  "Top-level Rust node types that get their own org section.")

(defconst literate-org-rust--use-like-types
  '(use_declaration extern_crate_declaration)
  "Rust top-level nodes that collapse into one \"Use declarations\" block.")

(defun literate-org-rust--expr-stmt-is-macro-call-p (node)
  "Non-nil if NODE is an `expression_statement' wrapping a `macro_invocation'."
  (and (eq 'expression_statement (tsc-node-type node))
       (> (tsc-count-children node) 0)
       (eq 'macro_invocation
           (tsc-node-type (tsc-get-nth-child node 0)))))

(defun literate-org-rust--doc-comment-p (node)
  "Non-nil if NODE is a Rust *doc* comment (=///= or =//!=).
Doc comments belong to the item that follows, so the use-grouping pass
must NOT swallow them — they should slide forward to the next item."
  (and (eq 'line_comment (tsc-node-type node))
       (string-match-p "\\`//[/!]" (tsc-node-text node))))

(cl-defmethod literate-org-next-code-block-index ((language (eql rust))
                                                  root start-index)
  (cl-block nil
    (let* ((max-index (1- (tsc-count-children root)))
           (idx start-index))
      (while (<= idx max-index)
        (let* ((node (tsc-get-nth-child root idx))
               (type (tsc-node-type node)))
          (cond
            ;; Use-like run — collapse contiguous use/extern_crate.
            ;; Plain line_comment is allowed as glue (e.g. `// re-exports`
            ;; between two `use` lines), but *doc* comments (=///= / =//!=)
            ;; belong to the next item and must NOT be swallowed here.
            ((memq type literate-org-rust--use-like-types)
             (let ((end idx))
               (while (and (< end max-index)
                           (let* ((nxt (tsc-get-nth-child root (1+ end)))
                                  (nxt-type (tsc-node-type nxt)))
                             (or (memq nxt-type literate-org-rust--use-like-types)
                                 (and (eq nxt-type 'line_comment)
                                      (not (literate-org-rust--doc-comment-p nxt))))))
                 (cl-incf end))
               (cl-return (cl-values end "Use declarations"))))
            ;; Real item — emit its title.
            ((memq type literate-org-rust--item-types)
             (cl-return (cl-values idx
                                   (literate-org-node-name language type node))))
            ;; expression_statement wrapping macro_invocation also counts.
            ((literate-org-rust--expr-stmt-is-macro-call-p node)
             (cl-return (cl-values idx
                                   (literate-org-node-name language type node))))
            ;; Prelude (attributes / comments / inner attributes / anything
            ;; we don't recognise) — slide forward; if we're already at the
            ;; end, emit it verbatim so the caller terminates.
            (t
             (if (= idx max-index)
               (cl-return (cl-values idx
                                     (literate-org-node-name language type node)))
               (cl-incf idx)))))))))
#+END_SRC

**** Terraform

Terraform configuration is HCL, and HCL's tree-sitter grammar differs
from Python / Rust / TypeScript in three ways that drive the design:

1. *Constructs nest one level deeper.* The parse tree is
   =config_file= → =body= → list of =block=, not =source_file= → list
   of items.
2. *Every top-level construct is the same node type — =block=.* A
   =resource=, =variable=, =output=, =module=, =data=, =locals=,
   =terraform=, =provider= are all =block= nodes; they differ only by
   their first =identifier= child and their =string_lit= labels. So
   there is no per-node-type dispatch table like Rust's — one
   =node-name= method reads the identifier + labels.
3. *Comments live at two levels.* A comment BEFORE the first block is
   a =config_file= child (sibling of =body=); a comment BETWEEN blocks
   is a =body= child. 18 of mega-code-infra's 34 =.tf= files open with
   a =#= header, and section dividers (=# ── EKS ──=) sit between
   blocks — all must round-trip. A naive "iterate =body='s children"
   loses the leading header.

Because of (3), Terraform overrides =get-code-blocks-in-current-buffer=
with a *cursor partition* rather than reusing the shared
node-by-node scanner: it slices the buffer at =block= boundaries so
every byte (leading header, between-block dividers, trailing comments,
blank lines) lands in exactly one section's body. That is the
byte-equivalence guarantee the shared scanner cannot give here.

HCL has no module/import system, so (like Rust) Terraform support is
pure import + tangle round-trip — the Python literate-server / loader
machinery is untouched.

***** file extension, parser language, babel name

Extension =.tf=. The tree-sitter grammar is =hcl= (covers Terraform),
so =treesit-parser-language= maps =terraform → hcl=. The babel block
language stays =terraform= (cosmetic + font-lock; we tangle, never
evaluate).

#+BEGIN_SRC elisp
(cl-defmethod literate-org-file-extension ((_language (eql terraform)))
  "tf")

(cl-defmethod literate-org-treesit-parser-language ((_language (eql terraform)))
  'hcl)

(cl-defmethod literate-org-babel-language-name ((_language (eql terraform)))
  "terraform")
#+END_SRC

***** node name — identifier + stripped labels

A =block='s heading is =<identifier> <label1>.<label2>= when it has
two labels (=resource aws_s3_bucket.asm=), =<identifier> <label1>=
with one (=variable region=), or just =<identifier>= with none
(=terraform=, =locals=). Labels are =string_lit= nodes whose text
*includes* the surrounding quotes, so they are stripped.

#+BEGIN_SRC elisp
(defun literate-org-terraform--block-labels (node)
  "The =string_lit= label texts of a HCL block NODE, quotes stripped."
  (cl-loop for i below (tsc-count-children node)
           for c = (tsc-get-nth-child node i)
           when (eq 'string_lit (tsc-node-type c))
           collect (string-trim (tsc-node-text c) "\"" "\"")))

(cl-defmethod literate-org-node-name ((_language (eql terraform))
                                      (_node-type (eql block)) node)
  (let* ((id-node (literate-org--child-of-type node 'identifier))
         (kind (if id-node (tsc-node-text id-node) "block"))
         (labels (literate-org-terraform--block-labels node)))
    (pcase labels
      ('() kind)
      (`(,one) (format "%s %s" kind one))
      (`(,a ,b . ,_) (format "%s %s.%s" kind a b)))))
#+END_SRC

***** get code blocks — cursor partition over block boundaries

Slices the buffer into one section per HCL =block=, with each
section's body running from the *end of the previous block* (or the
buffer start, for the first) to the *end of this block*; the last
section runs to =point-max=. This absorbs every leading header,
between-block divider, trailing comment, and blank line into exactly
one section's body — the partition is total, so the tangle round-trip
loses no content. (Inter-block blank-line spacing may still shift per
the org-babel padline rule that every language shares; =terraform fmt=
wired into the tangle path restores it.)

A file with no blocks (comments only, or empty) becomes a single
section holding the whole buffer. The module comment is the contiguous
=#= / =//= run at the very top, used only to title the file's section.

#+BEGIN_SRC elisp
(defun literate-org-terraform--leading-comment (root)
  "The contiguous run of comment text at the top of the file, from the
=config_file= ROOT's leading =comment= children, with =#=/=/= markers
stripped — or nil."
  (let ((count (tsc-count-children root)) (index 0) (parts nil) (stop nil))
    (while (and (not stop) (< index count))
      (let ((child (tsc-get-nth-child root index)))
        (if (eq 'comment (tsc-node-type child))
            (progn
              (push (string-trim-left (tsc-node-text child) "[#/ \t]+") parts)
              (cl-incf index))
          (setq stop t))))
    (when parts (mapconcat #'identity (nreverse parts) "\n"))))

(cl-defmethod literate-org-module-comment ((_language (eql terraform)) node)
  (literate-org-terraform--leading-comment node))

(cl-defmethod literate-org-get-code-blocks-in-current-buffer
    ((language (eql terraform)) module-name section-level)
  (let* ((language (literate-org-alias-language language))
         (tree (literate-org-parse-tree-for-current-file language))
         (root (tsc-root-node tree))
         (body (literate-org--child-of-type root 'body))
         (module-comment (literate-org-terraform--leading-comment root))
         (blocks (and body
                      (cl-loop for i below (tsc-count-children body)
                               for c = (tsc-get-nth-child body i)
                               when (eq 'block (tsc-node-type c)) collect c)))
         (code-blocks nil))
    (if (null blocks)
        ;; No blocks (comments only / empty) — one section, whole buffer.
        (setf code-blocks
              (list (make-literate-org-code-block
                     :title (or module-name "config")
                     :body (buffer-substring-no-properties (point-min) (point-max))
                     :language language :level section-level)))
      (let ((last-index (1- (length blocks)))
            (cursor (point-min)))
        (cl-loop for blk in blocks
                 for i from 0
                 for seg-end = (if (= i last-index)
                                 (point-max)
                                 (tsc-node-end-position blk))
                 do (setf code-blocks
                          (cons (make-literate-org-code-block
                                 :title (literate-org-node-name language 'block blk)
                                 :body (buffer-substring-no-properties cursor seg-end)
                                 :language language :level section-level)
                                code-blocks))
                    (setf cursor seg-end))
        (setf code-blocks (nreverse code-blocks))))
    (cons module-comment code-blocks)))
#+END_SRC

***** noweb split — deferred (follow-up)

A big =resource= / =module= could noweb-split on its nested =block=
members, but HCL's grammar delimits a block body with =block_start= /
=block_end= tokens wrapping a =body= node (not the =​{= / =}= child
tokens =literate-org--split-braced= assumes), so the shared brace
splitter does not apply directly. Deferred until a real 80+-line
single-resource pain point appears; without a =terraform= method,
=literate-org-split-node= returns nil and big blocks stay atomic —
the correct, safe default for the flat-import use case
(mega-code-infra).

***** Tests — block titles, byte-complete partition, tangle round-trip

The first three tests are pure parse assertions (titles for every block
shape; the cursor partition reproduces the source byte-for-byte even
with a leading header outside =body=; a comment-only file becomes one
section). The fourth is a real round-trip: import a =.tf= into an Org
buffer, =org-babel-tangle-file=, and assert the tangled output equals
the source *content* (non-blank lines — blank-line spacing shifts per
the shared org-babel padline rule).

The fifth is an optional real-world sweep: when
=LITERATE_ORG_TF_SAMPLE_DIR= names a directory of production =.tf=
files, every one must partition byte-complete. It is env-gated (never a

***** How to get the name of a parse node

The TSX dispatch table for =literate-org-node-name=. Vue SFC code
blocks fall through to TSX via =literate-org-alias-language= — even
though this subsection lives under =**** vue=, every =cl-defmethod=
dispatches on =(eql tsx)=. The =:kind= prefix on =lexical_declaration=
preserves =let= / =const= / =var= in the section title — losing it
silently merges three semantically different declarations into one
heading.

Same alist-pair-plus-catch-all shape as the Python and Rust
sections: five name-bearing nodes (function / type alias / enum /
interface / variable_declarator) plus three constants (string,
comment, import) fold into two =defconst=s; the three per-shape
methods (=export_statement= recurses into =:declaration= or =:value=,
=lexical_declaration= prepends =:kind=, =expression_statement=
unwraps) keep their own specialised dispatch.  =typescript= reuses all
of this verbatim: one =typescript= node-name method delegates to the
=tsx= dispatch, since the two grammars share these node shapes — only
=split-node= is registered per language, so without the delegation a
=typescript= block's titles fell through to the bare node-type.

#+BEGIN_SRC elisp
(defconst literate-org-tsx--name-labels
  '((function_declaration   . "Function")
    (type_alias_declaration . "Type")
    (enum_declaration       . "Enum")
    (interface_declaration  . "Interface")
    (variable_declarator    . "Variable"))
  "TSX node types whose section title is \"<label> <:name>\".")

(defconst literate-org-tsx--constant-labels
  '((string  . "String")
    (comment . "Comment")
    (import  . "Import"))
  "TSX node types whose section title is a fixed string.")

;; Catch-all: alist lookup; on miss, fall through to the generic default.
(cl-defmethod literate-org-node-name ((_language (eql tsx)) node-type node)
  (cond
    ((alist-get node-type literate-org-tsx--name-labels)
     (format "%s %s"
             (alist-get node-type literate-org-tsx--name-labels)
             (tsc-node-text (tsc-get-child-by-field node :name))))
    ((alist-get node-type literate-org-tsx--constant-labels))
    (t (cl-call-next-method))))

;; export_statement — recurse into :declaration if present, else show :value source.
(cl-defmethod literate-org-node-name ((language (eql tsx))
                                      (_node-type (eql export_statement)) node)
  (format "Export %s"
          (let ((declaration (tsc-get-child-by-field node :declaration)))
            (if declaration
              (literate-org-node-name language (tsc-node-type declaration) declaration)
              (let ((value (tsc-get-child-by-field node :value)))
                (if value
                  (tsc-node-text value)))))))

;; lexical_declaration — render as "<kind> <inner-name>" (e.g. "let foo").
(cl-defmethod literate-org-node-name ((language (eql tsx))
                                      (_node-type (eql lexical_declaration)) node)
  (format "%s %s"
          (tsc-node-text (tsc-get-child-by-field node :kind))
          (let ((sub-node (tsc-get-nth-child node 1)))
            (literate-org-node-name language
                                    (tsc-node-type sub-node)
                                    sub-node))))

;; expression_statement — recurse into the wrapped expression.
(cl-defmethod literate-org-node-name ((language (eql tsx))
                                      (_node-type (eql expression_statement)) node)
  (let ((new-node (tsc-get-nth-child node 0)))
    (literate-org-node-name language (ts-node-type new-node) new-node)))

;; typescript shares tsx's node shapes (function_declaration,
;; lexical_declaration, export_statement, etc.) and registers split-node
;; under `typescript', but had no typescript node-name method — so resplit
;; titles fell through to the bare node-type. Delegate every typescript
;; node-name to the tsx dispatch; the grammars share the same node fields.
(cl-defmethod literate-org-node-name ((_language (eql typescript)) node-type node)
  (literate-org-node-name 'tsx node-type node))

#+BEGIN_SRC elisp
(cl-defmethod literate-org-get-code-blocks-in-current-buffer ((language (eql vue))
                                                              module-name section-level)
  (let* ((language (literate-org-alias-language language))
         (tree (literate-org-parse-tree-for-current-file language))
         (root (tsc-root-node tree))
         (count-of-children (tsc-count-children root))
         (module-comment (when (> count-of-children 0)
                           (literate-org-module-comment language root)))
         (main-items nil)
         (comment-blocks nil)
         (script-blocks nil)
         (elements-blocks nil)
         (style-blocks nil)
         (code-blocks nil))
    (cl-loop for index from 0 below count-of-children
             for node = (ts-get-nth-child root index)
             for children-count = (tsc-count-children node)
             for node-type = (tsc-node-type node)
             for node-start-text = (when (> children-count 0)
                                     (tsc-node-text (tsc-get-nth-child node 0)))
             for node-end-text = (when (> children-count 0)
                                   (tsc-node-text (tsc-get-nth-child node (1- children-count))))
             for node-body-noweb-ref = nil
             for body = (if (> children-count 0)
                          (string-join (loop for i from 1 below (1- children-count)
                                             collect (tsc-node-text (tsc-get-nth-child node i))))
                          (tsc-node-text node))
             do (labels ((%build (node-type &key language 
                                            mode
                                            (literate-lang (symbol-name language))
                                            (indent 0))
                           (values
                            (format "%s<<%s-%s>>"
                                    (string-join (loop repeat indent collect " "))
                                    module-name node-type)
                            (with-temp-buffer
                              (insert body)
                              ;; (when mode
                              ;;   (funcall mode)
                              ;;   (prettier-prettify))
                              (cons 
                               (make-literate-org-code-block
                                :title (format "%s of %s" node-type module-name)
                                :properties (list (cons "literate-lang" literate-lang))
                                :level section-level)
                               (cdr
                                (literate-org-get-code-blocks-in-current-buffer
                                 language
                                 (format "%s-" module-name node-type)
                                 (1+ section-level))))))))
                  (ecase node-type
                    (script_element
                     (multiple-value-bind (noweb-ref blocks)
                         (%build "script" :mode 'typescript-ts-mode :language 'tsx)
                       (setf node-body-noweb-ref noweb-ref
                             script-blocks (nconc comment-blocks blocks))))
                    (element
                     (multiple-value-bind (noweb-ref blocks)
                         (%build "template" :mode 'vue-mode :language 'vue-html :indent 2)
                       (setf node-body-noweb-ref noweb-ref
                             elements-blocks (nconc comment-blocks blocks))))
                    (style_element
                     (multiple-value-bind (noweb-ref blocks)
                         (%build "style" :language 'css)
                       (setf node-body-noweb-ref noweb-ref
                             style-blocks (nconc comment-blocks blocks))))
                    (comment
                     (let ((block (make-literate-org-code-block
                                   :title (format "Comment %s" module-name)
                                   :body body
                                   :language 'vue
                                   :level (1+ section-level))))
                     (setf node-body-noweb-ref nil
                           comment-blocks
                           (nconc comment-blocks (list block)))))))
             (when node-body-noweb-ref
               (setf comment-blocks nil)
               (let ((node-text (format "%s\n%s\n%s\n"
                                        node-start-text
                                        node-body-noweb-ref
                                        node-end-text)))
                 (setf main-items (nconc main-items (list node-text))))))
    (setf code-blocks
            (cons (make-literate-org-code-block
                   :title (format "Main wrapper %s" module-name)
                   :body (string-join main-items"\n")
                   :language 'vue
                   :level section-level)
                  (nconc script-blocks elements-blocks style-blocks)))
    (cons module-comment code-blocks)))

  (interactive)
  (org-mark-subtree) ;; mark the current subtree
  (forward-line) ;; move point forward, so the headline isn't in the region
  (when (string-prefix-p ":PROPERTIES:" (thing-at-point 'line t))
    (search-forward ":END:")
    (forward-line))
  (delete-region (region-beginning) (region-end)) ;; delete the rest

         (cells (coerce (cdr (assoc 'cells data)) 'list))
         (current-org-level (org-current-level)))
    (dolist (cell cells)
      (let ((cell-type (cdr (assoc 'cell_type cell)))
            (source (string-join (coerce (cdr (assoc 'source cell)) 'list))))
        (cond ((string= cell-type "code")
               (if (string-prefix-p "!" source) 
                 (progn
                   (loop repeat (1+ current-org-level) do (insert "*"))
                   (insert " A shell command" "\n")
                   (insert "#+BEGIN_SRC " "sh" "\n")
                   (insert (subseq source 1))
                   (insert "\n#+END_SRC\n"))
                 (let ((src-file (make-temp-file "literate-org-ipynb-code" nil ".py"))
                       (language 'python))
                   (with-temp-file src-file
                     (insert source))
                   (let ((code-blocks
                          (with-current-buffer (find-file-noselect src-file)
                            (cdr (literate-org-get-code-blocks-in-current-buffer
                                  language "" 1)))))
                     (delete-file src-file)
                     (loop for block in code-blocks do
                       (loop repeat (literate-org-code-block-level block) do (insert "*"))
                       (insert " " (literate-org-code-block-title block) "\n")
                       (insert "#+BEGIN_SRC " (literate-org-babel-language-name (literate-org-code-block-language block)))
                       (if (loop for prefix in literate-org-ipynb-tangled-prefix-patterns
                                 never (string-prefix-p prefix (literate-org-code-block-title block)))
                         (insert " :tangle no"))
                       (insert "\n" (literate-org-code-block-body block))
                       (insert "\n#+END_SRC\n"))))))
              ((string= cell-type "markdown")
               (insert (literate-org-markdown-to-org source) "\n")
               (setf current-org-level (org-current-level))))))))

#+BEGIN_SRC python
import os
import django
from django.core.management import call_command
from threading import Thread

# Set up Django environment
os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'myproject.settings')
django.setup()

# Function to start the server
def start_server():
    call_command('runserver', '127.0.0.1:8000', '--noreload')

# Start the server in a new thread
server_thread = Thread(target=start_server)
server_thread.start()

print("Django server started on http://127.0.0.1:8000")
