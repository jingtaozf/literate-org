* Six families (mechanism-based)
:PROPERTIES:
:CUSTOM_ID: families
:END:

| Family | Members | Mechanism |
|--------+---------+-----------|
| In-context anchoring | A, E | Within-session output shaped by visible prior content |
| Pretraining prior return | D, E, I | Cross-session reset to RLHF baselines |
| Attention pattern bias | C, H | Architectural — positional bias, salience-without-verification |
| Persistence-stack dependence | B, J | Cross-session continuity entirely depends on durable artefacts |
| Multi-agent stigmergy | G | Coordination via shared artefact with zero agent-to-agent awareness |

* Eight cross-cutting observations
:PROPERTIES:
:CUSTOM_ID: observations
:END:

1. *Codification is the agent's only learning surface*. Dimensions
   that improve over time are exactly the dimensions codified in
   rules. Implicit-only conventions do not survive cross-session.
2. *Architectural phenomena define practical horizons*. A, C, F, H
   cannot be eliminated by rules; doctrine routes around them.
3. *The persistence stack is multi-purpose*. =tasks/todo.md=,
   =RATIONALE.md=, =ARCHITECTURE.org=, =tasks/lessons.md= each
   serve cold-start reconstruction (B), multi-agent coordination
   (G), and drift detection (J).
4. *Sycophancy compounds within sessions*. Single-turn
   measurements under-represent the cowork-relevant rate; long
   sessions show non-stationary drift.
5. *Convergent regression mechanism is pretraining attractor*.
   The dominant cause is the agent's pretraining distribution
   pulling toward generic patterns — not linguistic anchoring,
   not RLHF register preference.
6. *Lost in the middle has LP-specific implications*. Long .org
   files (>3000 lines) have systematic mid-content under-reading.
7. *Multi-agent emergence is pure stigmergy*. Current cowork has
   no inter-agent coordination layer above git.
8. *Capability heterogeneity is mostly invisible*. literate-agent
   doctrine is calibrated for Claude; other families produce
   different outcomes but the differences haven't been measured
   at scale.

* Postscript — the three-loop arc
:PROPERTIES:
:CUSTOM_ID: arc
:END:

Together with the prior two loops, this research describes /how
literate-programming artefacts function in human-agent
collaborative authorship/ at three scales:

- Loop 1 (=docs/transfer-gradient.org=): reader-author surface
  between artefact and consumer.
- Loop 2 (=docs/cowork-research.org=): dual-author interaction
  maintaining the artefact.
- Loop 3 (this file): agent-native mechanisms that make the
  dual-author interaction possible (or break it).

The gaps that remain — multi-agent coordination (G),
capability heterogeneity (F), long-horizon dynamics (I, J) — are
the research frontiers where the model is currently weakest. Each
is partly addressed by a new rule in this loop, but the
underlying empirical knowledge is thin and rests on this
project's own observation more than on published literature.
